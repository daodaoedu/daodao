# Proposal: 鼓勵語系統

## 背景

主題實踐的 email 目前有 PE01-PE08（建立確認、首次打卡鼓勵、週報、結業信），但**沒有「提醒你去打卡」的信**。設計稿中曾規劃 `daily-reminder`、`streak-warning` 等 job type，但從未實作。

目前已有 `checkin_encouragements` 表和 `encouragement.service.ts`，內含 30 句系統預設鼓勵語，在打卡成功後隨機顯示一句。但全是系統寫死的，沒有使用者投稿機制。

希望透過「人寫的鼓勵語」讓每封信帶有溫度和驚喜感。

## 現況盤點

```

──────────────────────────────    ──────────────────────────────
```

## 兩個互補的概念

### A. 社群鼓勵語池

邀請社群成員寫鼓勵語（可選擇署名或匿名），匯入公共鼓勵語池。每次寄信時隨機抽取一則，讓收信者感受到「有人在遠方默默為我加油」。

**核心體驗**：社群連結感、驚喜感

```
┌─────────────────────────────────────┐
│ 💬 來自社群的一句話：                 │
│ 「慢慢來，比較快。」                  │
│              —— 匿名島民              │
└─────────────────────────────────────┘
```

**可能的實作路徑**：擴充現有 `checkin_encouragements` 表，加入 `author_id`、`author_name`、`is_anonymous`、`status`（pending/approved）、`context_tag` 等欄位，讓它從「系統預設表」升級為「社群投稿池」。

### B. 寫信給未來的自己

讓使用者在特定時機寫一段話給未來的自己。當提醒信寄出時，帶上自己曾寫下的文字，像是「初心提醒」。

**核心體驗**：自我對話感、初心回顧

```
┌─────────────────────────────────────┐
│ 📝 第 1 天的你曾說：                  │
│ 「希望一個月後，靜坐已經變成           │
│   像刷牙一樣自然的事。」              │
└─────────────────────────────────────┘
```

**注意**：`practice_checkins.note` 是對外可見的打卡內容，不適合直接拿來當「寫給自己的話」。需要獨立欄位或表。

## 兩者的關係

A 和 B 不衝突，可以同時出現在一封信中。也可以分階段做。

```
提醒信結構（兩者並存時）
┌────────────────────────────────────────┐
│ Hi {name}，                             │
│ 你的實踐「{title}」已經進行到第 N 天了。 │
│                                        │
│ ┌────────────────────────────────┐     │
│ │ 💬 來自社群的一句話：            │     │
│ │ 「每一小步都算數。」             │     │
│ │            —— 小明              │     │
│ └────────────────────────────────┘     │
│                                        │
│ ┌────────────────────────────────┐     │
│ │ 📝 開始這趟旅程時你曾說：       │     │
│ │ 「希望一個月後，靜坐已經變成     │     │
│ │   像刷牙一樣自然的事。」        │     │
│ └────────────────────────────────┘     │
│                                        │
│ [去打卡]                                │
└────────────────────────────────────────┘
```

## 前置依賴

這個功能需要「提醒信」作為載體。目前提醒信不存在，有兩個選擇：

1. **先建提醒信，再加鼓勵語** — 完整但工程量較大
2. **先把鼓勵語加進現有的信（週報 PE03-PE06）** — 快速驗證效果，再決定是否建專屬提醒信

## 待釐清

### 社群鼓勵語池 (A)

- **寫入入口**：在哪裡邀請大家寫？獨立頁面？實踐頁內嵌？活動式徵集？
- **情境分類**：是否需要 tag（輕推 / 慶賀 / 里程碑）以避免語境錯置？
- **審核**：是否需要管理員審核後才入池？
- **署名**：署名顯示 display_name？還是自訂署名？匿名時顯示什麼（「匿名島民」）？
- **種子資料**：初期數量不足時，現有 30 句系統鼓勵語可直接作為種子

### 寫信給未來的自己 (B)

- **寫入時機**：建立實踐時？打卡時？獨立入口？
- **是否可編輯/刪除**：寫完後還能改嗎？
- **顯示邏輯**：每封信都帶同一則？還是寫了多則就輪流？
- **隱私**：這些內容只有自己看得到，對吧？

### 共通

- 鼓勵語是否只用在 email？還是也出現在 app 內（打卡成功對話框、首頁）？
- 優先做 A 還是 B？還是一起做？
- 優先掛載在哪種信上？（週報 vs 新建提醒信）
