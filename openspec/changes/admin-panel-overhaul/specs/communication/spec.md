# Communication 規格

## ADDED Requirements

### Requirement: 站內通知建立

Admin SHALL 能夠建立站內通知，包含標題、內容、及類型（公告／提醒／系統通知）。系統 MUST 驗證標題與內容不得為空。

#### Scenario: 建立一則公告通知

- **WHEN** Admin 填寫標題、內容，並選擇類型為「公告」後送出
- **THEN** 系統建立該通知並顯示於通知歷史列表中

#### Scenario: 未填寫必要欄位時阻擋送出

- **WHEN** Admin 未填寫標題或內容即嘗試送出
- **THEN** 系統 MUST 顯示驗證錯誤訊息，不得建立通知

### Requirement: 站內通知受眾選擇

Admin SHALL 能夠選擇通知發送對象：全部使用者、具有特定標籤的使用者、或指定個別使用者。

#### Scenario: 發送通知給全部使用者

- **WHEN** Admin 選擇受眾為「全部使用者」並送出通知
- **THEN** 系統將通知發送給所有使用者

#### Scenario: 發送通知給特定標籤的使用者

- **WHEN** Admin 選擇受眾為「特定標籤」並勾選一或多個標籤後送出
- **THEN** 系統僅將通知發送給具有所選標籤的使用者

#### Scenario: 發送通知給指定個別使用者

- **WHEN** Admin 選擇受眾為「指定使用者」並搜尋選取個別使用者後送出
- **THEN** 系統僅將通知發送給被選取的使用者

### Requirement: 站內通知排程發送

Admin SHALL 能夠排程通知在未來指定時間發送。

#### Scenario: 設定排程時間後建立通知

- **WHEN** Admin 建立通知時啟用排程功能，並設定未來的日期與時間
- **THEN** 系統將通知狀態設為「排程中」，並於指定時間自動發送

#### Scenario: 排程時間已過期時阻擋

- **WHEN** Admin 設定的排程時間早於目前時間
- **THEN** 系統 MUST 顯示錯誤訊息，不得建立排程通知

### Requirement: 站內通知歷史與統計

通知歷史頁面 SHALL 顯示所有已發送通知的列表，包含已發送數量、已讀數量、及已讀率。

#### Scenario: 檢視通知歷史列表

- **WHEN** Admin 進入通知歷史頁面
- **THEN** 頁面顯示所有通知，每則通知 MUST 顯示已發送數量、已讀數量、已讀率

#### Scenario: 通知發送後統計更新

- **WHEN** 一則通知已發送且部分使用者已讀
- **THEN** 該通知的已讀數量與已讀率 MUST 反映最新狀態

### Requirement: 站內通知預覽

Admin SHALL 能夠在正式發送前預覽通知的呈現效果。

#### Scenario: 預覽通知內容

- **WHEN** Admin 填寫通知內容後點擊「預覽」
- **THEN** 系統顯示通知在使用者端的呈現樣式，包含標題、內容與類型標示

### Requirement: 觸發式信件規則建立

Admin SHALL 能夠建立觸發式信件規則，包含觸發條件與信件動作。

#### Scenario: 建立一條觸發規則

- **WHEN** Admin 選擇觸發條件並設定對應的信件模板後儲存
- **THEN** 系統建立該規則並顯示於規則列表中

### Requirement: 觸發條件類型

觸發條件 SHALL 包含以下類型：沉睡 N 天、註冊 N 天未執行特定動作、標籤被新增或移除。

#### Scenario: 設定「沉睡 N 天」觸發條件

- **WHEN** Admin 選擇條件類型為「沉睡天數」並輸入天數 30
- **THEN** 系統將於使用者沉睡達 30 天時觸發該規則

#### Scenario: 設定「註冊 N 天未打卡」觸發條件

- **WHEN** Admin 選擇條件類型為「註冊天數未執行動作」，輸入天數 3，選擇動作為「打卡」
- **THEN** 系統將於使用者註冊滿 3 天且未打卡時觸發該規則

#### Scenario: 設定「標籤異動」觸發條件

- **WHEN** Admin 選擇條件類型為「標籤新增」並指定標籤名稱
- **THEN** 系統將於該標籤被加到使用者身上時觸發該規則

### Requirement: 信件模板編輯

Admin SHALL 能夠編輯信件模板，包含主旨、內文（富文本）、及變數（如 {{user_name}}、{{days_inactive}} 等）。

#### Scenario: 編輯信件模板並插入變數

- **WHEN** Admin 編輯信件模板，填寫主旨與內文，並在內文中插入 {{user_name}} 變數
- **THEN** 系統儲存模板，且實際發送時 MUST 將變數替換為對應使用者資料

#### Scenario: 預覽信件模板

- **WHEN** Admin 點擊模板預覽
- **THEN** 系統以範例資料替換變數後顯示信件呈現效果

### Requirement: 觸發規則啟用與停用

每條觸發規則 SHALL 具備啟用／停用切換功能。

#### Scenario: 停用一條啟用中的規則

- **WHEN** Admin 將一條啟用中的規則切換為停用
- **THEN** 系統停止該規則的觸發判斷，不再發送信件

#### Scenario: 重新啟用一條已停用的規則

- **WHEN** Admin 將一條已停用的規則切換為啟用
- **THEN** 系統恢復該規則的觸發判斷

### Requirement: 觸發信件執行紀錄

頁面 SHALL 顯示執行紀錄，記錄每封信件的發送對象、發送時間、是否被開啟、是否被點擊。

#### Scenario: 檢視執行紀錄

- **WHEN** Admin 進入觸發信件的執行紀錄頁面
- **THEN** 頁面顯示所有已發送信件的列表，每筆 MUST 包含收件者、發送時間、開啟狀態、點擊狀態

### Requirement: 觸發規則彙總統計

頁面 SHALL 顯示每條規則的彙總統計，包含總發送數、開啟率、點擊率。

#### Scenario: 檢視規則統計

- **WHEN** Admin 在規則列表頁面檢視某條規則
- **THEN** 該規則 MUST 顯示總發送數、開啟率、點擊率
