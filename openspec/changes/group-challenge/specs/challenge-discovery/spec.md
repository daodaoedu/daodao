## ADDED Requirements

### Requirement: 首頁挑戰卡片顯示
系統 SHALL 在每位登入使用者的首頁自動顯示當前最新一個非「已結束」狀態的共同挑戰卡片。卡片 SHALL 標示此為共同挑戰（有別於個人挑戰），並顯示名稱、期間、當前狀態標籤。

#### Scenario: 已登入使用者看到挑戰卡片
- **WHEN** 使用者登入後進入首頁
- **THEN** 頁面顯示共同挑戰卡片，卡片上有「共同挑戰」標示、挑戰名稱、起迄日期、狀態標籤

#### Scenario: 無進行中挑戰時首頁不顯示卡片
- **WHEN** 系統中沒有狀態為「報名中」、「未開始」或「進行中」的挑戰
- **THEN** 首頁不顯示共同挑戰卡片

### Requirement: 挑戰卡片狀態標籤
挑戰卡片 SHALL 根據挑戰當前狀態顯示對應標籤：
- `draft`（草稿）：顯示「草稿」，僅管理員可見
- `enrolling`（報名中）：顯示「報名中」
- `upcoming`（未開始）：顯示「即將開始」
- `active`（進行中）：顯示「進行中」
- `ended`（已結束）：顯示「已結束」

#### Scenario: 報名中狀態卡片
- **WHEN** 挑戰狀態為 `enrolling`
- **THEN** 卡片顯示「報名中」標籤，未參加者顯示「立即加入」按鈕

#### Scenario: 已參加者看到不同按鈕
- **WHEN** 使用者已報名，且挑戰狀態為 `upcoming`
- **THEN** 卡片顯示「未開始」，不顯示「立即加入」按鈕

#### Scenario: 挑戰結束後卡片移入過往紀錄
- **WHEN** 挑戰狀態切換為 `ended`
- **THEN** 首頁主要入口不再顯示此卡片，可於使用者個人頁的過往挑戰區塊找到

### Requirement: 彈挑視窗（Pop-up）
系統 SHALL 在使用者登入後，若存在狀態為 `enrolling` 或 `upcoming` 的挑戰，顯示一次彈挑視窗。彈挑視窗 SHALL 對所有登入使用者顯示（包括已參加者），因為挑戰可重複參加。

#### Scenario: 登入後出現彈挑視窗
- **WHEN** 使用者登入，且當前有 `enrolling` 狀態的挑戰
- **THEN** 顯示彈挑視窗，內含挑戰資訊、當前參與人數、剩餘報名時間

#### Scenario: 無符合條件挑戰時不顯示彈挑視窗
- **WHEN** 使用者登入，但沒有 `enrolling` 或 `upcoming` 狀態的挑戰
- **THEN** 不顯示彈挑視窗

#### Scenario: 同一 session 中彈挑視窗只出現一次
- **WHEN** 使用者在同一 browser session 中已關閉過彈挑視窗
- **THEN** 重新整理頁面後不再彈出（以 sessionStorage 記錄）

### Requirement: 即時數據顯示於彈挑視窗與卡片
彈挑視窗與挑戰卡片 SHALL 顯示「剩餘報名時間」與「當前參與人數」，數據 SHALL 透過 API 即時取得。

#### Scenario: 顯示剩餘報名時間
- **WHEN** 挑戰狀態為 `enrolling`，使用者查看卡片或彈挑視窗
- **THEN** 顯示報名截止前剩餘天數或小時數

#### Scenario: 顯示當前參與人數
- **WHEN** 使用者查看卡片或彈挑視窗
- **THEN** 顯示已報名的參與者總人數（即時從 API 取得）
