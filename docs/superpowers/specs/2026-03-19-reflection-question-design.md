# Reflection Question Feature Design

**Date:** 2026-03-19
**Status:** Approved

## Overview

在打卡表單中新增「反思問題」功能。使用者可透過開關啟用，開啟後顯示一個隨機反思問題，引導使用者寫出更有深度的打卡紀錄，而非流水帳。

## User Flow

1. 使用者開啟打卡 Sheet
2. 在「想法分享」區塊，可看到「反思問題」開關（預設關閉）
3. 開啟後，描述欄上方顯示一張問題卡片
4. 使用者看完問題後，在描述欄中作答
5. 不喜歡這題可點「換一題」，隨機切換（排除當前這題）
6. 問題卡片不影響表單提交

## Design Decisions

- **問題不填入描述欄**：問題只作為視覺提示，使用者自己決定如何作答
- **前端固定**：25 題寫死在前端，不需後端，維護簡單
- **開關預設關閉**：跟引導句開關行為一致
- **隨機排除當前題**：換一題不會連續出現同一題

## Architecture

### New Files

**`hooks/use-reflection-question.ts`**
管理問題清單、當前題目索引、換題邏輯。

```ts
// 回傳
{ question: string, nextQuestion: () => void }
```

換題邏輯：從 25 題中隨機選，排除當前 index。

**`components/reflection-question.tsx`**
顯示問題卡片 + 換一題按鈕。接收 `question` 和 `onNext` props，無狀態。

### Modified Files

**`check-in-sheet.tsx`**
在「想法分享」區塊加入 `<ReflectionQuestion />` 元件，放在 `<TagSelector />` 和 `<DescriptionField />` 之間。

### Component Layout

```
想法分享
├── TagSelector（含引導句開關）
├── ReflectionQuestion（含反思問題開關，預設關閉）
│   └── [開啟後] 問題卡片 + 換一題按鈕
└── DescriptionField
```

## UI Spec

問題卡片樣式：淺色背景（`bg-gray-50`）、圓角、`border`，左側顯示問題文字，右側「換一題 →」按鈕。

開關樣式：與引導句開關一致（Tailwind toggle）。
