# AI 引導旅程設計規劃

> 靈感來源：AI agent 引導用戶逐步完成任務，讓用戶感覺「這是我自己做到的」，而非「AI 幫我做的」

---

## 問題背景

### 核心痛點

DaoDao 用戶在使用 Practice 與 Challenge 時面臨兩個關鍵障礙：

1. **不知道如何規劃** — 有想法但不知道怎麼拆解成可執行的步驟
2. **懶惰紀錄** — 打卡這件事有四層摩擦：太麻煩、不知道記什麼、記了沒感覺、忘記

### 為什麼現在做

- Challenge 已有群體框架，但缺乏個人化節奏引導
- Practice 完全手動設定，沒有任何 AI 輔助
- 分享機制已存在，但缺乏值得分享的「成就時刻」

---

## 設計目標

| 目標 | 衡量指標 |
|------|---------|
| 用戶能在 3 分鐘內從「想做某事」到「有明確的今日任務」 | 首次任務設定完成率 |
| 打卡從填空變成確認 | 每日 check-in 完成率提升 |
| 完成里程碑後用戶主動分享 | 成就卡分享率 |

### 目標用戶感受

完成一段旅程後，用戶說的不是「AI 幫我規劃好了」，而是 **「我做到了！」** 並且想讓別人看見。

---

## 設計方向

兩條並行的產品線，共用相同的核心組件：

```
Challenge + AI 引導    ←──┐
                          ├── 核心組件（AI 規劃 + 引導 check-in + 成就分享）
Practice + AI 引導     ←──┘
```

---

## Phase 1：Challenge + AI 引導

### 場景

用戶加入一個官方 Challenge（例如：30 天英文挑戰）後，AI 幫他把挑戰個人化，產生符合自己節奏的任務計畫。

### 用戶旅程

```
加入挑戰
   ↓
AI 快問（3 題以內）
   ↓
生成個人化任務節奏
   ↓
每日推播：「今天這件事做了嗎？」
   ↓
一鍵確認（可選：補充心情/圖片）
   ↓
里程碑達成 → 成就卡 → 分享
```

### AI 快問設計（加入挑戰後觸發）

最多 3 個問題，用選擇題降低摩擦：

1. **你每天能投入多少時間？** — 15 分鐘 / 30 分鐘 / 1 小時以上
2. **你偏好在什麼時段執行？** — 早上 / 下班後 / 睡前 / 隨機
3. **你目前的程度？**（視挑戰類型動態調整）

### 引導式 Check-in

取代現有空白表單，改為：

- **主要動作**：「✓ 今天完成了『[任務名稱]』」一鍵確認
- **次要動作**（可選）：心情標籤、圖片、一句話
- **推播時機**：根據用戶設定的偏好時段觸發

### 成就時刻

- **週里程碑**：7 天連續打卡 → 「你已經連續 7 天了！」動畫 + 可分享卡片
- **挑戰完成**：全程完成 → 成就卡包含：天數、打卡次數、一句 AI 生成的個人化總結
- **分享卡格式**：視覺化的旅程時間軸 + 成就標語

## Phase 2：Practice + AI 引導

### 場景

用戶自己建立 Practice（或從模板建立）時，AI 介入規劃流程，將模糊目標拆解為具體任務。

### 用戶旅程

```
建立新 Practice（或選擇模板）
   ↓
輸入目標：「我想學吉他」
   ↓
AI 引導對話（4-5 題）
   ↓
生成建議任務清單 + 時間節奏
   ↓
用戶確認 / 微調
   ↓
進入日常 check-in 循環（同 Phase 1 機制）
   ↓
里程碑達成 → 成就卡 → 分享
```

### AI 引導對話（建立 Practice 時）

1. **目標**：你想達成什麼具體結果？（ex. 能彈一首完整的歌）
2. **時間**：你想在多久內達成？（1 個月 / 3 個月 / 半年以上）
3. **現況**：你目前的起點是？（完全新手 / 有一點基礎 / 想提升到新水平）
4. **節奏**：你能投入的頻率？（每天 / 每週幾次）

AI 根據回答生成：
- 建議的里程碑（每 2-4 週一個）
- 每次 check-in 對應的任務提示
- 預估完成時程

### 與 Challenge 的差異

| | Challenge | Practice |
|---|---|---|
| 框架來源 | 官方提供 | AI 從目標生成 |
| 任務固定程度 | 相對固定（個人化節奏） | 高度客製化 |
| 社交屬性 | 公開 + 群體能量 | 個人為主，可選公開 |
| AI 複雜度 | 低（個人化已有框架） | 高（從零規劃） |

---

## 核心組件（兩 Phase 共用）

### 1. AI 規劃引擎

- 輸入：目標描述 + 用戶偏好（時間、節奏、程度）
- 輸出：里程碑清單 + 任務建議 + 推播時程
- 設計原則：AI 提建議，用戶最終確認——保留掌控感

### 2. 引導式 Check-in 元件

- 取代現有空白 check-in 表單
- 主動顯示「今日任務是什麼」
- 完成動作最小化（一鍵確認優先）
- 支援補充（心情、圖片、文字）

### 3. 成就卡（擴充現有 summary card）

- 現有基礎：`practice-summary-card.tsx` + `use-practice-summary-image.tsx` + 分享到 LINE / Threads / IG 等多平台
- 新增內容：Challenge 完成場景支援 + AI 生成個人化總結文字（例如「你堅持了 28 天，這不是一件小事」）
- 不需要從零建，只需擴充現有元件

### 4. 成長地圖（長期）

- 個人頁面的視覺化旅程軌跡
- 展示完成的 Practice / Challenge 時間軸
- 讓用戶「看見自己的積累」

---

## 執行順序

```
Phase 1a：Challenge AI 快問 + 任務個人化        ← 最小可驗證
Phase 1b：引導式 Check-in（取代空白表單）
Phase 1c：成就卡 + 分享機制
───────────────────────────────────────────
Phase 2a：Practice 建立時的 AI 引導對話
Phase 2b：Practice 引導式 Check-in
Phase 2c：成長地圖
```

---

## 非目標（此次不做）

- 不修改 Challenge 的社交分層邏輯（已有設計）
- 不自動執行任務（AI 建議，用戶決定）
- 不做跨用戶的任務比較或排名
- 成長地圖留到 Phase 2c，不在 Phase 1 範圍內

---

## 決策記錄

1. **AI 規劃引擎** → 呼叫現有 LLM API，不微調
2. **成就卡分享** → 擴充現有 `practice-summary-card`，支援外部分享（LINE、Threads、IG 等）
3. **AI 快問入口** → 整合在 Challenge 詳情頁，不是加入後的彈窗
