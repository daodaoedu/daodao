# 信號層規劃：如何呈現使用者的累積與能力

> 承接 `docs/product/prd/learning-ecosystem.md` 的「信號層」（生態圈的果實）。
> 回應的痛點：目前不知道怎麼呈現使用者的累積與能力；ESCO 標準化技能標籤嘗試後因難度擱置。
> 校準日期：2026-07-06。資料資產以當日程式碼盤點為準。

## 一、為什麼會卡：三個互斥的約束

「呈現累積與能力」之所以難，是因為同時被三個約束夾住，而它們幾乎無法用單一「等級／分數」同時滿足：

1. **分類法陷阱**——想把混亂的自主學習塞進標準技能詞彙（ESCO）。維護成本極高（龐大對照表＋中英標籤），且讓使用者有「被系統評分歸類」的感覺，違反心理安全原則。
2. **憑證陷阱**——「能力」隱含一個**平台無法驗證的勝任聲明**。島島不是發證機構；顯示「日文 Lv.3」等於做假證，使用者也不會信。
3. **島島自己的指標禁令**——連續天數、學習時長、deadline 達成率全部禁用（製造壓力）。連「累積」最直觀的量化解法都被自己擋掉。

**卡點的本質：想同時「量化能力」又「不做假證」又「不製造壓力」，這三件事沒辦法用一個等級或分數同時達成。** ESCO 只是這個死結的其中一次嘗試，不是問題本身。

## 二、核心原則：呈現「投入的證據」，不呈現「能力等級」

一個轉念同時鬆開三個約束：

> **平台不評斷使用者的能力，只把「他投入了什麼、被誰看見」攤開，讓能力的判斷交給觀看者。**

- 繞開分類法陷阱：不需要標準詞彙。
- 繞開憑證陷阱：不宣稱勝任，只展示做過什麼。
- 不製造壓力：呈現的是歷程的質地，不是耐力。

這正是島島「連接器、不是內容提供者」哲學的延伸——平台不打分，只讓學習被看見。

## 三、把「累積」與「能力」拆成兩條線

混在一起想是卡住的主因之一。兩者資料來源、給誰看、風險都不同：

| | 累積（你做了什麼） | 能力（你能做什麼） |
|---|---|---|
| 本質 | 歷程的**質地** | 歷程被**見證**的程度 |
| 主要給誰看 | 本人（動機／留存） | 他人（信任／配對／發現） |
| 資料來源 | check-ins、practices、outcomes、projects、tags | reactions、複製次數、被閱讀數 |
| 風險 | 低（攤開自己的事） | 中（要避免變排行榜／假證） |

## 四、三層呈現方案

三層由下而上、由「本人看」到「他人看」、由低風險到需要更多資料累積。全部可從既有資料建，不需要 ESCO。

### 第 1 層：累積 = 足跡（Footprint）

**目的**：本人回訪時看見「我累積了什麼」，對抗自主學習的隱形努力感。
**呈現形式**：
- **領域雲**：聚合 practice／check-in 的自由標籤，字級＝投入程度。標籤自然浮現，不需預定義軸。
- **近期節律**：近 7 天（或近 N 次）的實踐分佈——刻意用「近期」而非「連續」，繞開 streak 禁令。
- **產出牆**：把 outcomes／projects 攤成作品格。這是累積最有重量的證據。

**資料來源（皆現有）**：`daodao-server` 的 `tag.routes.ts`（含 popular tags）、`practice.routes.ts`、`outcome.routes.ts`、`project.routes.ts`（`GET /projects/outcomes/me`）、check-in 資料。
**繞開哪個約束**：全部——只攤開本人的事，無等級、無 streak、無標準分類。
**與既有 PRD 的關係**：這是 `docs/product/island`（我的小島）的具體化。**但 island PRD 的「成長雷達圖」要重新考慮**：雷達圖需要預定義的軸（哪幾種能力？），一畫軸就又踩回分類法陷阱。建議改用領域雲（標籤驅動、無需預定義軸），或把雷達軸從「能力面向」改成「投入面向」（如探索／實作／反思／分享的比例），後者衡量的是行為分佈而非能力高低，不做假證。

### 第 2 層：能力 = 被見證的證據（Witnessed Evidence）

**目的**：他人（Buddy、連結對象、圍觀者）快速判斷「這個人在做什麼、值不值得連結」。
**呈現形式**：不自我宣稱，用他人的質化反應背書。例如「這個人在『日文』的實踐，被 12 人標記為有洞察、被複製 5 次」。
**能力＝社群給的信號，不是平台的評分，也不是本人的宣稱**——三個約束全繞開。
**資料來源（皆現有）**：六種 reaction（`reaction.routes.ts`，其中 useful≈Insightful）、複製次數（`practice.routes.ts` 的 copy 來源記錄）、被閱讀（f2e view-tracking）。這三者正對應島島既定的三個質化指標 Insightful／Referenced／Witnessed。
**風險與守則**：
- 不做全站排行榜（會製造比較壓力、贏者全拿）。信號綁在「某人的某領域」的脈絡裡，不抽出來做跨人排名。
- 冷啟動保護：反應數少時顯示質性描述（「有人覺得有洞察」）而非赤裸數字，避免「0」的挫折——沿用 inspire-feed 的冷啟動處理精神。
- 先等第 1 層上線、質化互動資料累積一段時間再做，否則畫面全是空的。

### 第 3 層：驗證章 = 有門檻的收成（Verified Milestone）

**目的**：學習者的「結業式」與對外可攜帶的信譽。
**呈現形式**：只對「已完成且有品質」的實踐蓋章，門檻沿用 `docs/product/practice` 資產化 PRD（≥14 天且 3 個洞察驗證）。
**誠實的聲明範圍**：驗證章只證明「完成了一段有見證的歷程」，**不宣稱「精通某技能」**——這是它能繞開憑證陷阱的關鍵。文案上務必避免暗示能力等級。
**資料來源**：第 1、2 層的資料 + 實踐完成狀態 + 資產匯出（`docs/product/practice` 的匯出 FRD，PDF／Markdown）。
**位置**：這是信號層最上層、也是最自然的付費點（對齊經濟層——見生態圈分析的「收費點對齊生態價值」）。放最後做。

## 五、標籤策略（ESCO 的替代品）

不做標準分類，改「**自由標籤 ＋ 熱門標籤收斂**」：
- 讓標籤自然長出來（practice／check-in 已支援自由標籤，`tag.routes.ts` 已有 popular tags）。
- 需要顯示「相關領域」時，用標籤 co-occurrence（哪些標籤常一起出現）做鬆散聚類，不強制對到標準詞彙。
- 真要幫使用者整理時，用 AI 做「**建議標籤**」而非「強制分類」——AI 當助手，不當裁判。

這條路犧牲了跨使用者的標準可比性（無法保證兩個人的「日文」是同一個標準節點），但換來零維護成本、零心理壓力、可漸進。對自主學習平台，這個取捨划算。若未來真需要標準化（如對外履歷、跨平台認證），再局部引入映射，而非一開始就全域標準化。

## 六、呈現面（畫面落點）

| 呈現面 | 放哪幾層 | 對應既有規劃 |
|--------|----------|--------------|
| 我的小島（個人檔案） | 足跡（給本人）＋ 被見證（給訪客） | `docs/product/island` |
| 展示廣場 / 搜尋 | 被見證（幫助發現與判斷） | `docs/product/search` |
| 匯出作品集 | 驗證章 + 足跡快照 | `docs/product/practice`（資產化／匯出） |
| Buddy / 連結決策點 | 被見證（判斷要不要連結） | `docs/product/buddy`、`docs/product/social` |

## 七、分期

1. **Phase 1 — 足跡**：領域雲＋近期節律＋產出牆。資料全現成、對本人有即時動機、風險最低。同時把 island PRD 的雷達圖決策重新定案（改領域雲或改投入面向軸）。
2. **Phase 2 — 被見證**：待質化互動資料累積後，在個人檔案與展示廣場加「被見證的證據」。同步確認 reaction→質化指標的聚合邏輯。
3. **Phase 3 — 驗證章**：接資產匯出與完成門檻，作為信號層最上層與付費點。

## 八、成功指標（沿用生態圈的指標哲學：量流動、不量壓力）

- **足跡回訪**：使用者查看自己足跡頁後 7 天內是否再次打卡（動機是否被點燃），而非「頁面停留時間」。
- **被見證覆蓋率**：有多少比例的活躍實踐至少獲得 1 個質化信號（能量是否流動到收成層）。
- **驗證章轉化**：完成門檻的實踐中，有多少實際匯出作品集（收成是否被珍惜）。
- **禁用**：任何「累積排行」「能力分數分佈」「完成度百分比壓力條」——會把展示變成競賽。

## 九、開放問題（待拍板）

- **island 雷達圖去留**：改領域雲、改投入面向軸、還是兩者並存？（Phase 1 前必須定案）
- **被見證的數字露出程度**：顯示精確數字（12 人）還是質性描述（有人覺得有洞察）？涉及比較壓力，建議冷啟動期用質性、成熟後可切數字。
- **標籤是否需要輕度收斂**：完全自由 vs 提供熱門標籤讓使用者選，影響領域雲的可讀性與可聚合性。
