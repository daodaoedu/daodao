# 島島阿學 Email 通知設計文件

## 核心設計理念

島島阿學的 Email 通知系統以「社交輔助學習」為核心目標，而非單純的訊息推播。每一封通知都應該服務於以下三個學習動力：

1. **被看見** — 有人關注你的學習旅程，強化繼續分享的動力
2. **看見他人** — 夥伴正在行動，觸發同儕效應與互相牽引
3. **相互問責** — 學習夥伴的存在，創造最強的持續動力

> 通知不是打擾，是學習社群的連結信號。

---

## 一、每日互動 Digest（4 小時批次）

### 社交學習角色

**「有人與你互動、夥伴在前進」的複合型通知** — 將過去 4 小時內的社交信號彙整成一封有層次的信件，讓用戶感受到「學習社群的溫度」，同時透過學習夥伴的動態觸發問責感。

### 機制說明

- **執行時間**：每日 08:00 / 12:00 / 16:00 / 20:00（UTC+8）
- **涵蓋範圍**：過去 4 小時的通知事件；早班（08:00）回看 12 小時，補抓夜間缺口
- **排除項目**：`wish.linked`（即時寄送）
- **個人化**：依 `notification_preferences (N01)` 尊重用戶個別設定，未設定預設開啟

### Email 結構（依社交學習價值排序）

#### 🤝 你的學習夥伴有動態（最高優先）

**社交學習設計意圖**：學習夥伴是最強的問責機制。當你知道夥伴正在行動，最自然的反應是「我也該去做了」。

| 包含事件 | 說明 |
|---------|------|
| `PartnerCheckinActivity` | 學習夥伴完成了一次打卡 |
| `PartnerUpdateActivity` | 學習夥伴更新了主題實踐 |

- 卡片樣式：橘色左邊框（醒目，區別於一般互動）
- CTA 文字：「去看看他的動態 →」

---

#### 💬 有人與你互動

**社交學習設計意圖**：有人注意到你、回應你，強化「分享是值得的」這個正向循環。讓用戶從被動學習者變成社群的一份子。

| 包含事件 | 說明 | Priority |
|---------|------|---------|
| `comment` | 有人在你的內容留言（practice / checkin / **persona_answer**） | P1 |
| `comment_reply` | 有人回覆了你的留言 | P1 |
| `mention` | 有人在留言中 @ 你 | P1 |
| `Connect` | 有人想與你連結 | P1 |
| `ConnectAccepted` | 對方接受了你的連結邀請 | P1 |
| `BuddyRequest` | 有人想成為你的學習夥伴 | P1 |
| `BuddyAccepted` | 對方接受了你的夥伴邀請 | P1 |
| `PracticeCreated` | 你關注/連結的人開始了新實踐 | P1 |

- 卡片樣式：青色左邊框
- CTA 文字：「前往查看 →」

---

#### 🌱 你關注的人有新動態

**社交學習設計意圖**：看見他人在做什麼，創造同儕學習效應。輕量呈現，不搶奪注意力，但保留「去探索」的入口。

包含其餘 P1 activity 類事件（`UserFollowActivity`、`ConnectRequestActivity` 等），以簡潔列表呈現，附 inline「查看 →」連結。

---

#### ✨ 共鳴回饋

**社交學習設計意圖**：Follow 和 Reaction 都是一種認同，讓用戶知道自己的學習旅程正在被人關注，強化長期分享動力。

| 包含事件 | 說明 | Priority |
|---------|------|---------|
| `reaction` | 有人對你的實踐、打卡或**人物誌回答**按了 reaction（CTA → `/persona/{question_id}`） | P2 |
| `UserFollowed` | 有人開始追蹤你 | P2 |
| `PracticeFollowed` | 有人追蹤了你的主題實踐 | P2 |

---

### 主旨動態邏輯

| 情境 | 主旨 |
|------|------|
| 只有夥伴動態 | `【島島阿學】{name}，你的學習夥伴有新動態！` |
| 只有互動 | `【島島阿學】{name}，有人與你互動了` |
| 只有共鳴 | `【島島阿學】{name}，有人對你的內容送出了共鳴` |
| 混合 | `【島島阿學】{name}，你有 {N} 則新動態` |

### Header 文案動態邏輯

| 情境 | 文案 |
|------|------|
| 只有夥伴動態 | 你的學習夥伴正在前進，去看看他們在做什麼，或許能給彼此一些動力！ |
| 只有互動 | 有人注意到你的學習旅程，快去看看吧！ |
| 夥伴 + 互動 | 你的島嶼熱鬧起來了！有夥伴與你互動，也有學習夥伴在前進。 |
| 其他 | 你的島嶼有新動態，快去看看吧！ |

## 二、每週社群 Digest（週批次）

### 社交學習角色

**「溫暖陪伴的週節奏」** — 在週期結束時，讓用戶感受到「這週我有在學習，也有人陪著我一起走」。重點不是數字，而是連結感與被看見的溫暖。

### 機制說明

- **執行時間**：每週日 20:00（UTC+8）
- **涵蓋範圍**：UTC+8 週一 00:00 ～ 週日 23:59 的所有通知事件
- **個人化**：依 `notification_preferences (N01)` 過濾，未設定預設開啟

### Email 內容

#### 有人看見了你（溫暖散文式，非數字方塊）

將本週社交互動以一段溫暖文字呈現，而非四格大數字儀表板。

| 數據 | 來源事件 |
|------|---------|
| 被 reaction 數 | `reaction` 事件 |
| 被留言 / 回覆數 | `comment` + `comment_reply` 事件 |
| 新追蹤者數 | `UserFollowed` + `PracticeFollowed` 事件 |
| 連結邀請數 | `Connect` 事件 |

範例文案：「這週有 8 個共鳴、3 位新追蹤者，謝謝你持續分享自己的學習旅程。」

若本週無任何互動，不顯示此區塊。

#### 你這週的學習足跡

只顯示本週**有打卡或已完成**的實踐（最多 5 個），不顯示未開始的實踐，避免製造愧疚感。

- 實踐名稱
- 本週打卡次數
- 狀態：`進行中` / `已完成`

#### 你的學習夥伴這週也在前進（若有學習夥伴）

顯示學習夥伴正在進行的實踐，以陪伴而非比較的方式呈現。

- 夥伴名稱 + 正在探索的實踐名稱
- CTA：「去為他加油 →」

### 主旨

溫暖陪伴式，而非報告書式。

| 情境 | 主旨 |
|------|------|
| 有實踐進度 | `【島島阿學】{name}，島島陪你回顧這一週 🌿` |
| 無實踐但有互動 | `【島島阿學】{name}，這週有人看見了你的學習` |
| 預設 | `【島島阿學】{name}，島島陪你回顧這一週 🌿` |

## 事件優先級總覽

依 PRD 定義，通知事件分三個優先級：

| Priority | 定義 | Email 處理方式 |
|---------|------|--------------|
| **P1** 高價值 | 需要回應的互動：連結、留言、@、夥伴動態、關注的人開始實踐 | 進入 4 小時 Digest，P1 section |
| **P2** 共鳴性 | 被動認同：Follow、Reaction | 進入 4 小時 Digest，共鳴回饋 section |
| **P3** 學習摘要 | 週期性回顧 | 每週日週報 |

---

## 用戶控制

所有 Email 通知均支援：
- **分項設定**：用戶可針對每種通知類型開關 Email（`notification_preferences` 表，channel `N01`）
- **退訂連結**：每封 Email footer 含「取消訂閱此類通知」直達連結
- **通知設定連結**：每封 Email 含「調整通知設定」連結
