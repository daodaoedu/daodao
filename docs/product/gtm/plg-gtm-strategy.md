# 島島阿學 PLG GTM 策略：從 Civic Tech 到可持續生態圈

> 版本：v1（2026-07）
> 範圍：3 個月內建立可預測的 Free → Paid 轉換漏斗，獲取首批付費用戶
> 依據：八層生態圈現況盤點、daodao-storage 訂閱 schema 現況（僅 `subscription_plan` / `user_subscription` 兩張表，無金流、無權限 enforcement）

---

## Executive Summary

1. **先量測、再收成、才收費**：3 個月分三階段走——Phase 0（第 1–4 週）用已通電的主循環建立生態儀表板與付費意願驗證（創始會員預售），Phase 1（第 5–10 週）以「資產匯出」作為信號層 MVP 啟動首批付費，Phase 2（第 11 週起）用複製鏈與展示廣場建立零廣告的自增長引擎，並瞄準 2027 年 2 月學期窗口啟動實驗教育機構 B2B pilot。
2. **收費點只落在信號層與引導層的「收成時刻」**（資產匯出、數位驗證章、AI 額度），養分循環的全部環節（實踐、打卡、回應、靈感牆、複製、治理）永久免費，且**不對任何既有免費功能回頭收費**。
3. **既有 civic tech 社群不是轉換障礙而是轉型的正當性來源**：用治理層（許願池、Roadmap 投票）讓社群共決付費點順序與定價，以「創始會員」與「島民貢獻計畫」把最忠誠的免費用戶變成第一批付費者與傳教士。

## 1. 價值鏈就緒度評估表

「付費就緒」的判準：這一層是否已經產生**用戶願意用錢交換的收成**，而不是功能是否存在。

| # | 生態層 | 通電度 | 付費就緒度 | 商業角色 | 前置條件（若要承載付費） |
|---|--------|--------|-----------|----------|--------------------------|
| 1 | 個體層（實踐 CRUD、打卡、複製） | ~100% | ❌ 永久免費 | 漏斗頂端 + 養分循環起點 | 無——收費會直接殺死生態 |
| 2 | 引導層（AI action-maker、quiz、onboarding） | ~70% | 🟡 可分級（AI 額度） | Basic/Premium 差異化 | **Phase 0 前置**：AI 呼叫用量計數器（目前無 per-user quota 機制） |
| 3 | 同儕層（快速回應、留言、關注、連結、Buddy） | ~70% | ❌ 永久免費 | 養分循環的回流引擎 | 無。⚠️ Buddy 僅有請求收發、無配對後體驗——**半成品不得列入任何付費方案** |
| 4 | 可見性層（靈感牆、Social Hub、展示廣場） | ~70% | ❌ 永久免費 | 動機引擎 + 自然獲客面（SEO/分享） | 無。展示廣場個人頁自訂屬「裝飾」，依原則不收費 |
| 5 | 信號層（資產匯出 PDF/MD、數位驗證章） | ~10% | 🔴 未就緒，但是**核心付費點** | Paid 的主要價值載體 | **Phase 1 前置**：資產匯出 MVP（先做，工程量小）→ 數位驗證章（後做，需信任機制設計） |
| 6 | 節律層（通知、推薦、週報、workflow） | ~50% | 🟡 部分（深度週報/洞察） | Premium 加值 | **Phase 1–2 前置**：學習週報引擎（`docs/product/weekly-report` 已有規劃） |
| 7 | 治理層（許願池、Roadmap 投票） | ~100% | ❌ 永久免費 | **轉型正當性工具**——社群共決付費點的場域 | 無 |
| 8 | 經濟層（訂閱系統） | ~10% | 🔴 未就緒 | 收銀台 | **Phase 1 前置**：金流串接（藍新/綠界/TapPay 擇一）、plan enforcement middleware（daodao-server）、admin 訂閱管理（daodao-admin-ui）。目前只有 DB schema |

**審計結論（決定整份策略的形狀）：**

- 目前**沒有任何已上線體驗足以直接支撐訂閱收費**。信號層 10%、經濟層 10%，意味著 Phase 0 在產品內收不到錢——這不是壞消息，而是把 Phase 0 的任務定義清楚：**建儀表、養預期、收訂金（場外）**。
- 已通電的主循環（打卡 → 靈感牆 → 快速回應 → 回流）是**免費飛輪**，它的健康度是付費轉換的先行指標，不是付費對象。
- 最短收費路徑是**資產匯出**：資料都在（打卡的心情/筆記/照片/標籤），工程上是渲染問題不是資料問題；數位驗證章需要「驗證」的信任設計（誰授予、依據什麼），排在匯出之後。

## 2. 付費 Persona 卡片

共同判準：他們付錢買的不是內容（島島沒有內容），是**連接產生的收成**——行動軌跡被組織、被證明、被看見的那一刻。

### P1 ｜轉職者「阿哲」（B2C，付費潛力最高）

| 欄位 | 內容 |
|------|------|
| 輪廓 | 27–34 歲，離開第一份工作自學軟體/設計/行銷，3–9 個月轉職窗口 |
| 痛點 | 自學軌跡散落在 Notion、HackMD、GitHub，面試時說不出「我這半年做了什麼」的可信故事 |
| 為什麼為「連接」付費 | 島島把他的打卡行動**連接成一條可對外出示的軌跡**：匯出 PDF 學習歷程附在履歷後、驗證章連結放在 LinkedIn/CakeResume。他買的是「行動 → 雇主信任」這條連接，內容是他自己產的 |
| 決策者 / 預算 | 本人；轉職期焦慮預算高，NT$200–400/月 無痛（對照：求職者願花數千上 Hahow 課） |
| 收成時刻 | 面試前一晚匯出學習歷程 → **付費牆放在這裡** |
| 主通路 | Threads 學習帳號生態、PTT Soft_Job/Tech_Job、Dcard 工作板 |

### P2 ｜自學家庭「小羽與媽媽」（B2C，付費者與使用者分離）

| 欄位 | 內容 |
|------|------|
| 輪廓 | 國高中非學校型態實驗教育（自學）學生 + 主導教育決策的家長。全台個人自學生約 8–9 千人，加機構/團體實驗教育逾 2 萬人 |
| 痛點 | 自學計畫要向縣市審議會提交**學習狀況報告書**，家長每學期為「把孩子的學習整理成體制看得懂的文件」痛苦一次 |
| 為什麼為「連接」付費 | 島島把孩子日常打卡**連接成審議會格式的學習歷程文件**——買的是「自學日常 → 體制證明」的翻譯層。這是島島 DNA 最深的客群（島島源自自學社群），信任成本趨近於零 |
| 決策者 / 預算 | 家長付費、學生使用；自學家庭年教育支出數萬起，NT$1,500–3,500/年 的文件化工具在「報告書季」是剛需價格 |
| 收成時刻 | 每學期報告書提交前（**5–6 月、11–12 月**，季節性極強）|
| 主通路 | FB 自學/實驗教育家長社團、實驗教育推動中心、自學社群口碑、LINE OA 承接 |

### P3 ｜學習型創作者「Kai」（B2C，量少但是通路本身）

| 欄位 | 內容 |
|------|------|
| 輪廓 | 22–30 歲，在 IG/Threads 經營學習帳號（studygram、讀書帳），把學習過程當內容 |
| 痛點 | 「今天讀了什麼」的貼文產製累；學習軌跡沒有一個可以掛在 bio 的家 |
| 為什麼為「連接」付費 | 島島把他的實踐**連接成可分享的公開頁面**（展示廣場 + 匯出精美卡片/MD），買的是「學習行動 → 受眾內容」的生產線。他的每次分享都是島島的獲客素材 |
| 決策者 / 預算 | 本人；價格敏感，Basic 級（NT$149/月）為主，重點不是他的 ARPU 而是他的擴散係數 |
| 收成時刻 | 完成一個主題實踐、想發「成果文」的時候 |
| 主通路 | Threads / Instagram（台灣 Threads 滲透率全球前段，學習系帳號生態成熟）|

### P4 ｜實驗教育機構「拾光學苑」（B2B，Phase 2 主攻）

| 欄位 | 內容 |
|------|------|
| 輪廓 | 團體/機構型實驗教育單位、自學共學團，每單位 15–80 名學生，3–10 位導師 |
| 痛點 | 導師要同時看見每個孩子的學習節律，且每學期要產出給家長與審議會的成果文件；現用工具是 Google 表單 + 手工 |
| 為什麼為「連接」付費 | 買的是**機構視角的連接**：導師儀表板（看見全班的打卡節律與同儕網絡，不是監控分數）、批次匯出全班學習歷程、機構署名的數位驗證章（機構為學生的實踐背書 → 解決島島「誰來驗證」的信任問題，**機構本身成為生產者多樣性的解方**）|
| 決策者 / 預算 | 機構負責人/教學主任；學期制預算，**決策窗口 8 月（新學年）與 2 月（下學期）**；私立/實驗教育經費彈性高於公立，免公家採購流程 |
| 收成時刻 | 學期末成果發表 + 審議文件產出 |
| 主通路 | 直接 outreach（各縣市非學校型態實驗教育審議通過名單為公開資料）、實驗教育年會/雜學校展會、自學社群轉介 |

**刻意不做的 persona**：企業內訓（島島無課程內容、無 SCORM/合規追蹤，與 HR 採購邏輯錯位，Phase 2 結束前不碰）；公立學校（公家採購 + 標案週期 > 12 個月，等機構線跑通再說）。

## 3. 分階段 GTM 路線圖

> 起算：2026-07 中。總長 12 週 + B2B 延伸線。

### Phase 0 ｜信號層未完成期（W1–W4）：建儀表、養預期、收訂金

**原則**：產品內一毛錢都收不到（經濟層 10%），所以 Phase 0 的「銷售」全部發生在場外，且每個動作都在為 Phase 1 鋪軌。

| 行動 | 內容 | 成功判準 | 失敗判準 |
|------|------|----------|----------|
| A0-1 生態指標儀表板 | 在 daodao-admin-ui 實作六大生態指標（資料已齊：checkin / quick_response / comment / follow / link / copy 表），先給內部看 | W2 結束前六指標有數字、有週趨勢 | W4 仍無基線 → Phase 1 延後，**不得盲飛上線收費** |
| A0-2 付費點公投 | 在既有許願池/Roadmap 投票丟出三個付費候選功能（匯出/驗證章/AI 額度），讓社群投「你願意為哪個付錢」——治理層變成 pricing research | ≥150 有效投票、匯出或驗證章得票 >40% | 投票 <50 → 社群溝通量能不足，先修復溝通管道再推轉型公告 |
| A0-3 創始會員預售 | 場外金流（綠界/Portaly 皆可）預售「創始島民」年票 NT$1,200/年（= Basic 年繳 83 折 + 終身創始徽章 + 定價治理投票權），限量 100，明示「資產匯出 9 月上線，不滿意 30 天全退」 | 4 週售出 ≥30 | <10 → 付費意願證偽，回頭檢查 P1/P2 假設，暫停 Phase 1 金流投資 |
| A0-4 收成預告 UX | 在打卡流程尾端加一行「這則紀錄已收進你的學習資產（9 月起可匯出）」——零功能、純文案，開始建立「打卡=累積資產」的心智 | 上線即成功（成本≈0） | — |
| A0-5 儀表板北極星確認 | 用 A0-1 數據確認主循環真的在轉：回應率、能量回流率取得基線 | 取得 4 週基線 | 回應率 <15% → 先做回應密度（見 Phase 0 備註）再談收費 |

**工程前置（同步跑）**：金流串接調研與串接（建議綠界，訂閱定期定額成熟、對小額月扣友善）、daodao-server plan enforcement middleware 設計、資產匯出 MVP 開發（目標 W8 上線）。

### Phase 1 ｜信號層 MVP 上線後（W5–W10）：首批付費轉換

| 行動 | 內容 | 成功判準 | 失敗判準 |
|------|------|----------|----------|
| A1-1 資產匯出 MVP 上線（W8） | PDF（履歷/審議會兩種版型）+ Markdown 匯出；免費用戶可**預覽全文但匯出上鎖**——收成看得見、帶走要付費 | 上線 2 週內 ≥25% 活躍用戶點過預覽 | 預覽率 <8% → 收成價值假設錯誤，訪談 10 位用戶再定 |
| A1-2 訂閱系統上線（W8） | 綠界定期定額 + Free/Basic/Premium 三級生效（Enterprise 手動開通）；創始會員自動升級 | 金流跑通、退訂自助 | — |
| A1-3 轉型公告 | 依第 6 節 Playbook 全通路發布（站內 + LINE OA + FB + Threads） | 公告後 7 日流失（週活跌幅）<10% | 週活跌 >20% → 啟動 Playbook 危機劇本 |
| A1-4 首批轉換 campaign | 對「近 90 天有 ≥5 次打卡」的用戶推「你已累積 N 則紀錄，匯出你的第一份學習歷程」分眾訊息（LINE OA + email） | Free→Paid 轉換 ≥3%（此群體） | <1% → 付費牆位置或定價錯，A/B 價格前先 A/B 牆的位置 |
| A1-5 報告書季搶灘（P2） | 11–12 月是自學報告書季：9–10 月在自學家長社團發「用島島產出審議會報告書」教學文 + 免費模板 | 帶入 ≥100 個自學家庭註冊、其中 ≥10% 付費 | 註冊 <30 → 通路選錯，改走實驗教育推動中心合作 |
| A1-6 B2B 種子 | 從自學社群人脈找 2–3 家機構免費 pilot（換 logo、案例、需求訪談），為 2 月窗口備料 | 2 家簽 pilot MOU | 0 家 → B2B 延後一學期，資源全押 B2C |

### Phase 2 ｜穩定期（W11 起）：可重複的獲客引擎（零廣告）

| 引擎 | 機制 | 為什麼可重複 |
|------|------|--------------|
| 複製鏈飛輪 | 匯出的 PDF/分享頁腳註「複製這個實踐開始你的版本」→ 每一份被帶去面試/發到 Threads 的收成品都是帶 CTA 的獲客素材 | 付費行為（匯出）本身生產獲客素材，CAC 隨付費量下降 |
| 展示廣場 SEO + 分享卡 | 公開實踐頁做 OG image（自動生成學習軌跡卡），Threads/IG 轉貼即有視覺物料 | P3 創作者的日常發文變成常態曝光 |
| 季節引擎 | 自學報告書季（5–6 月、11–12 月）× 轉職季（農曆年後、畢業季）排年度 campaign 日曆 | 需求是週期性剛需，每季重複收割 |
| B2B 學期引擎 | **9–11 月接觸 → 12–1 月試用 → 2 月簽約**（下學期窗口）；**3–5 月接觸 → 6–7 月試用 → 8 月簽約**（新學年窗口）。pilot 機構的學期末成果發表 = 下一輪機構的 demo | 台灣學期制讓 B2B pipeline 有固定節拍 |
| 驗證章網絡（Phase 2 後段，**前置：驗證章功能**） | 機構/導師授予的驗證章掛在學生對外檔案上 → 看到章的人點進來 → 機構成為信任節點 | 每個 B2B 客戶自動成為 B2C 獲客節點 |

## 4. 四級方案功能矩陣

**鐵律**：對收成收費、不對裝飾收費。免費版必須保有完整養分循環（打卡→被看見→被回應→回流），否則靈感牆枯萎、生態死亡、付費版也沒東西可賣。

| 功能 | Free | Basic NT$149/月（年繳 1,490） | Premium NT$349/月（年繳 3,490） | Enterprise NT$1,500/學生席/學期（20 席起，導師席免費） | 定價邏輯 |
|------|------|------|------|------|----------|
| 實踐 CRUD、打卡、複製 | ✅ 無限 | ✅ | ✅ | ✅ | 養分循環起點，收費即斷循環 |
| 快速回應、留言、@mention、關注、連結 | ✅ 無限 | ✅ | ✅ | ✅ | 回流引擎，免費用戶的回應是付費用戶的動機來源 |
| 靈感牆、Social Hub、展示廣場 | ✅ | ✅ | ✅ | ✅ | 被看見=動機+獲客面，鎖了漏斗頂端就沒了 |
| 許願池、Roadmap 投票 | ✅ | ✅ | ✅ | ✅ | 治理層是轉型正當性，永久免費 |
| AI action-maker / quiz | ✅ 每月 10 次 | ✅ 每月 50 次 | ✅ 不限量（合理使用） | ✅ 席次共用池 | 引導層按邊際成本分級——收的是 AI 成本，不是功能稅 |
| **資產匯出**（PDF 履歷版/審議會版、MD） | 🔍 全文預覽、每年 1 次免費匯出 | ✅ 每月 3 次 | ✅ 不限 + 自訂版型 | ✅ 全班批次匯出 | **核心收成點**。每年 1 次免費讓所有人嚐到收成，付費買的是頻率與場合 |
| **數位驗證章**（Phase 2 前置：功能開發） | 可獲得、可展示 | 可獲得、可展示 | ✅ 對外驗證連結（LinkedIn/履歷嵌入） | ✅ 機構署名授予權 | 章本身免費（榮譽不賣錢），**對外驗證的連接**才收費 |
| 學習週報 | ✅ 基礎版 | ✅ 基礎版 | ✅ 深度洞察版（前置：週報引擎） | ✅ 導師視角班級節律報告 | 節律層基礎免費（維繫回流），洞察收費 |
| 導師儀表板、團體空間 | — | — | — | ✅ | B2B 專屬，看節律與網絡、不看排名與時長 |
| 創始徽章 / 定價投票權 | — | 創始會員限定 | 創始會員限定 | — | 稀缺性給早鳥，不常售 |

**兩個刻意的免費決策**：
1. **每年 1 次免費匯出**——讓 P2 自學家庭在第一個報告書季免費體驗完整收成，第二季才付費；收成過的人才知道收成值多少錢。
2. **回應、靈感牆全免費且無廣告**——免費用戶不是「未轉換的付費用戶」，他們是生態的土壤；免費用戶的快速回應就是付費用戶留下來的原因。

## 6. 既有社群轉化 Playbook

### 6.1 敘事框架：「生態圈要自己養活自己」——三幕劇

- **第一幕（我們一起蓋的）**：回顧 civic tech 時期到現在，社群一起蓋了什麼——用數字說（實踐數、打卡數、回應數）。所有權敘事：「這是你們的島」。
- **第二幕（誠實的帳本）**：公開維運成本（infra、AI API、人力），揭露「靠愛發電」的不可持續。civic tech 社群尊重透明勝過完美——**公開帳本是這個社群的語言**。
- **第三幕（順應生態的答案）**：不是「開始收費」，而是「生態圈長出經濟層」——引用島島自己的哲學：先有果實，才有東西可賣。收費點由許願池投票共決（A0-2 已做，公告時引用投票結果：「你們選了資產匯出」）。

**關鍵語法**：永遠說「經濟層通電了」，不說「推出付費方案」；永遠說「收成收費」，不說「進階功能解鎖」。

### 6.2 轉型公告模板（站內 + LINE + FB + Threads 同步）

> **島島要長出經濟層了**
>
> 三年前，島島是一群自學生和夥伴在 civic tech 社群種下的種子。今天，島上有 __ 個實踐、__ 次打卡、__ 次彼此的回應。
>
> 誠實地說：維持這座島每個月要花 __ 元（帳本在這裡 →連結）。我們不想靠廣告，也不想賣你們的資料——所以我們選擇讓生態圈自己結果。
>
> 上個月許願池的投票裡，你們把「資產匯出」排在第一。9 月起，它上線：
> - **你現在擁有的一切，永遠免費。** 打卡、實踐、靈感牆、回應——一項都不會鎖。
> - 新的收成功能（把你的學習軌跡匯出成履歷附件、審議會報告書）採訂閱制，**每人每年 1 次免費匯出**。
> - 前 100 位「創始島民」：NT$1,200/年、終身創始徽章、以後所有定價你都有一票。
> - 學生與經濟困難者：申請「島民貢獻計畫」，用社群貢獻換訂閱。
>
> 這不是島島變了，是島島活下來的方式。帳本每季公開，方向永遠在 Roadmap 上讓你投票。

### 6.3 FAQ（預先發布，不等炎上）

| 問題 | 回答要點 |
|------|----------|
| 我原本用的功能會被鎖嗎？ | **不會，一項都不會。** 收費只落在新的收成功能上（匯出、對外驗證、AI 額度提升）。白紙黑字寫進公告 |
| 島島還開源嗎？ | 是，repo 不變、License 不變。開源專案收訂閱費有大量先例（open core），程式碼自由 ≠ 服務免費 |
| 學生 / 沒收入怎麼辦？ | 島民貢獻計畫：每月穩定的社群貢獻（有品質的回應陪伴、開源 PR、案例授權）可兌換 Basic；另有學生半價 |
| 為什麼不募款就好？ | 捐款養不出穩定維運（帳本為證），而且我們想證明「學習連接」本身值得付費——這比施捨更尊重這個生態 |
| 錢會用去哪？ | 每季公開帳本；營收先償付 infra 與 AI 成本，再投入 Roadmap 上票數最高的功能 |
| 我不付費會變二等島民嗎？ | 不會。靈感牆不分免費付費、回應不分、治理投票不分。免費用戶是島的土壤，不是待轉換名單 |

### 6.4 過渡期方案

| 對象 | 方案 | 用意 |
|------|------|------|
| 近 6 個月有打卡的既有用戶 | 自動獲得 Basic 6 個月（含每月 3 次匯出） | 最忠誠的人第一個嚐到收成，而不是第一個被要錢 |
| 創始會員（限量 100） | NT$1,200/年 + 終身徽章 + 定價投票權 | 把「反對收費的最大聲量」轉成「有股份的共建者」 |
| civic tech 時期貢獻者（code/設計/內容） | 終身 Basic | 償還社群債，公開名單致謝 |
| 島民貢獻計畫 | 月貢獻審核制兌換 Basic（防灌水：人工月審 + 上限名額） | 維持「用參與換取」的 civic 精神，同時防止回應灌水污染生態指標 |

### 6.5 危機劇本（週活跌 >20% 或社群強烈反彈時）

1. 48 小時內創辦人具名直播/長文回應，不用品牌帳號擋。
2. 不撤回收費（撤回=承認背叛敘事），但**開一場治理層緊急提案**：讓社群投「哪個收費點該調整」，把對抗轉回共決。
3. 逐一私訊迴響最大的 10 位老用戶——civic tech 社群的輿論由節點帶動，修復節點就修復輿論。

## 7. 統一指標 Dashboard 規格

**架構**：三層漏斗——生態健康（先行）→ 收成事件（中介）→ 商業結果（落後）。生態指標惡化必然在 4–8 週後反映到商業指標，反之商業指標不可用生態代價換取。

**戒律（硬性）**：禁用連續打卡天數、學習時長、嚴格 deadline 達成率——量流動不量壓力、量網絡不量個人耐力。以下所有指標均已審查合規。

| 層 | 指標 | 定義 | 計算方式（資料來源） | Phase 0 目標 |
|----|------|------|----------------------|---------------|
| 生態 | 回應率 | 公開打卡在 72h 內獲得 ≥1 個快速回應/留言的比例 | `checkin` × `quick_response`/`comment` join，週滾動 | 建立基線；基線 <15% 觸發回應密度工程 |
| 生態 | 能量回流率 | 收到回應的用戶 7 日內再次打卡的比例 | 回應事件 → 後續 checkin 存在性 | 基線 ×1.2（Phase 1 結束前） |
| 生態 | 複製鏈長度 | 實踐複製樹的中位深度 | `copy` 關係遞迴查詢 | 基線；Phase 2 起 ≥1.5 |
| 生態 | 連結密度 | 雙向連結數 ÷ 週活躍用戶數 | `link` 表 / WAU | 基線持平即可（轉型期不跌就是贏） |
| 生態 | 生產者多樣性 | 當月建立新實踐的用戶佔月活比例 | distinct practice creators / MAU | 基線；B2B pilot 後看機構是否拉升 |
| 生態 | Buddy 存活率 | 配對後 4 週雙方皆仍有打卡的比例 | ⚠️ **待功能**：Buddy 僅請求收發，配對後體驗未建——列入儀表但標記 N/A | N/A（誠實留空） |
| 收成 | 收成觸達率 | 活躍用戶中看過匯出預覽的比例 | 匯出預覽事件 / WAU | Phase 1：≥25% |
| 收成 | 收成完成數 | 匯出成功次數（免費+付費） | 匯出事件 | Phase 1：≥200 次 |
| 收成 | 收成→付費率 | 用完免費匯出額度者 30 日內升級比例 | 額度耗盡事件 → 訂閱事件 | Phase 1：≥10% |
| 商業 | 創始會員售出 | 場外預售數 | 金流後台 | Phase 0：≥30／失敗線 <10 |
| 商業 | MRR | 月經常性收入（年繳÷12 攤提） | `user_subscription` × 金流 | Phase 1 結束：≥NT$15,000 |
| 商業 | Free→Paid 轉換率 | 高活躍 free（90 天 ≥5 打卡）的付費比例 | cohort 查詢 | Phase 1：≥3% |
| 商業 | Churn（月） | 月退訂率 | 金流 webhook | <5%（樣本小，先看退訂原因質性） |
| 商業 | CAC | 獲客成本（無廣告 → 內容/活動工時折算） | 手工記帳 | 維持趨近 0，Phase 2 起監控 B2B 親售成本 |
| 商業 | LTV | ARPU ÷ churn | 上兩項推導 | Phase 2 起才有意義，Phase 1 不設目標 |
| 交叉 | 付費者回應貢獻比 | 付費用戶發出的快速回應佔比 | quick_response × subscription join | **哨兵指標**：若付費者只收成不回應 → 生態被抽血，檢討方案設計 |

**Phase 0 唯一北極星**：回應率 × 能量回流率（主循環是否在轉）。**Phase 1 北極星**：收成→付費率（收費點是否真的落在收穫時刻）。
