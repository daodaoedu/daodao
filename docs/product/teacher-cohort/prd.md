## 詞彙定案（2026-07-21 第三輪修訂，全產品面統一）

**結構詞與品牌詞分離**（2026-07-21 定案）：合作方光譜含 KOL、工作坊、社群主、讀書會、一對一諮詢——「課程」「營」對其中一半不合身。系統介面用中性結構詞，品牌詞退到行銷層：

| 層 | 詞 | 說明 |
|---|---|---|
| 結構詞（系統/UI） | **系列**（Program）、**期**（Cohort） | 「開新的一期」「本期動態」「本期名單」。「期」不預設形態——營隊、諮詢包、讀書會一輪都是一期，且本來就是市場計量單位（NT$4,900/期） |
| 品牌詞（行銷） | **陪跑營** | 僅用於 GTM 對團體營場景的行銷語言；系統不替使用者決定名字——`display_name` 自由文字（晨光寫作營、四月陪跑計畫、第五輪共讀皆可），同 Circle 的 Space 模式 |
| 不受影響 | 燈塔、教練、學員 | 燈塔無形態假設；學員為諮詢市場通用（實查）；介面以關係表達（「你的期」）少用角色名 |

> 本檔以下章節寫於「營」詞彙時期：結構語境的「營」一律讀作「期」，「陪跑營」僅於行銷語境保留。

### 第一、二輪詞彙紀錄（2026-07-16）

**刻意不用學校語言（班級/老師/學生），改用陪跑市場的慣用語**。詞彙依 2026-07-16 對台灣陪跑類服務的實查定案（樣本：AAPD 職涯陪跑計畫、媽媽商學院理財陪跑、雷蒙復盤陪跑班、閱讀前哨站陪跑營、Hahow 官方陪跑服務等）——市場共識是「營／教練／學員」：

| 用 | 不用 | 說明 |
|---|---|---|
| **陪跑營**（簡稱「營」） | 班級、課程、團 | 容器的產品名。「營」是群體＋固定期間場景的市場主流（陪跑營/挑戰營/訓練營）；「計畫」偏一對一顧問式；「團」非此市場用語，保留給 GTM 既有的島島自辦「衝刺團」 |
| **教練** | 老師、導師 | 營的發起者在島島的角色名，市場一致用法。「講師」只用於指稱其**站外身分**（課程講師）與 GTM 客群 |
| **學員** | 學生 | 參與者。「學員」是教練市場的通用詞、不帶學校味（與「學生」是兩回事），連最教練導向的服務都用它 |
| **開營** | 開班、開課 | 建立陪跑營 |
| **營內動態牆** | 班級牆 | 營內成員共同的動態牆 |
| **燈塔**（Lighthouse） | 管理後台、admin、partner、coach | 開營者管理端的產品名（2026-07-21 定案）。取島嶼意象：燈塔不拉著船走，只持續發光——與「陪跑不監控」同構。URL `/lighthouse` |

框架規則：**教練在營內動態牆上與學員平權**——都是打卡與回應，不是公告欄；教練不是站在講台上的人，是一群人一起衝刺時負責帶的那個人。這個框架落實到所有介面文案與互動設計。

## 核心概念

給課程講師的**陪跑教練台，不是 LMS**。島島的定位是學習連接器，核心有機質是主題實踐（practice）打卡。教練台的價值不是上傳教材、出考卷、算成績，而是：

**讓教練看見學員課後的學習節律，並在對的時機回應。**

這直接承接 GTM「B 軌」策略（`docs/product/gtm/plg-gtm-strategy.md`）：講師的痛點是完課率與續購，島島賣的是「陪跑空間」——講師把課程作業做成實踐模板 → 課程學員複製模板加入陪跑營、開始打卡 → 教練看全營節律 → 期末批次匯出成果當招生素材。

## 使用者問題

**教練端（站外身分為課程講師，付費者）**：
- 課程結束後學員練習斷裂，完課率低、續購難
- 看不見學員課後有沒有真的在做，只能在群組裡乾喊
- 招生缺乏「學員真實成果」的素材

**學員端**：
- 課後練習孤立無援，沒有節律、沒有回饋
- 注意：這群人是**被帶進來的使用者**（動機外部），與島島現有自願用戶行為模式不同——設計必須考慮轉化橋（見「北極星指標」）

## 設計原則（不可退讓）

1. **量流動、不量壓力**（承島島指標戒律）：儀表板呈現流動、引導回應；**永不**呈現缺勤排名、連續天數、學習時長、deadline 達成率。「需要關心名單」只呈現流動中斷＋該學員最後一則打卡，行動按鈕是「送個鼓勵」不是「發提醒」。
2. **營內容預設不進公共靈感牆**：只進營內動態牆，學員可自選公開——保護原生社群純度，也是隱私邊界。
3. **教練看到的是「營內的產出」，不是「學員這個人」**（見下節可見性模型）。

## 資料可見性模型

可見性的錨點是**練習與營的綁定關係**，不是使用者。推導鏈：

```
教練 ──(cohort_members.role = owner)──▶ 陪跑營(cohort)
營  ──(cohort_practices 綁定)────────▶ 學員的 practice
practice ──(隸屬關係)───────────────▶ 打卡(check-ins)
```

**可見性 = 「owner 身分」×「營綁定」×「關係存續」三者同時成立**，任一失效即不可見。

### 資料分界

| 教練看得到 | 教練看不到 |
|---|---|
| 學員名單（暱稱、頭像、加入時間） | 學員 email、真實姓名等個資 |
| 綁定本營的 practice 及其打卡與心得 | 學員自己另外開的任何 practice |
| 營內動態牆內的留言、回應互動 | 學員在其他營裡的打卡 |
| 本營聚合統計（節律熱度、標籤分佈） | 個人足跡、人物誌、追蹤/連結關係 |

### 規則細節

- **綁定發生在加入流程**：學員經邀請連結加入 → 複製營的模板 → 寫入 `cohort_practices`。同意畫面一句話：「你在本營建立的這個練習，其打卡內容將對教練可見」。未走此流程的 practice 永不在鏈上。
- **單則打卡「僅自己」：緩議**（2026-07-20 決策，推翻 07-16 的 Phase 1 定案）：本階段歸屬營的打卡對**教練＋全體同營學員**可見（因同營動態牆定案為完整可見），無單則隱藏選項。兩個強制配套：(a) 加入同意文案必須明講「對教練與同營學員可見」，不能只寫「提供給活動方」；(b)「僅自己」列為**第一批營結束後參與者訪談的必問題**——牆使可見範圍從一位教練擴大到整個營，自我審查的風險升高，此題必須拿真實回饋再定。原設計（勾選＋計入存在隱藏內容）保留於此供屆時採用。
- **自助退出／轉為個人實踐**（2026-07-20 定案，本階段即做）：學員可自行把活動實踐轉回個人實踐，等同退出該營的可見範圍，綁定解除即生效。加入靠同意、退出也必須自助——不能靠客服。
- **與社群可見性正交**：practice 既有的 `privacy_status`/`visibility` 管「對社群」，營可見性是另一軸，互不覆寫。營綁定絕不使資料外漏至公共靈感牆。
- **聚合統計走同一條鏈**：任何查詢不允許以 user_id 為起點撈 practice，一律從 `cohort_practices` join。AI 助教（Phase 3）餵給 ai-backend 的資料集同樣受此限制。

### API 三道防線（對照 daodao-server 既有模式）

1. **路由範圍化**：所有教練端 endpoint 掛在 `/api/v1/cohorts/:cohortId/*` 底下，middleware `requireCohortRole('owner')`（比照 `requireOwnership`）。不存在以人為單位的教練端 API。
2. **物件級歸屬檢查（防 IDOR）**：細節頁除驗 owner 外，必驗「該打卡所屬 practice 確實綁定本營」。測試須明確覆蓋跨營 ID 替換的 case。
3. **回傳欄位白名單**：Zod response validator 嚴格白名單（學員只回 `id/nickname/avatar/joined_at`）。email 永不給教練——通知學員走通知系統。

### 生命週期：三層資料、三種存續（2026-07-16 修正）

教練對服務對象的資料分三層，各有不同存續——**教練需要的是「服務紀錄」，不是「持續監看」**，任何服務業者都保有客戶名單，但內容是學員的、離營後的人生更是：

| 層 | 內容 | 存續 |
|---|---|---|
| **名冊層** | 暱稱、頭像、參加過哪幾期、完成/退出狀態 | **永久**——教練的營運紀錄，跨期服務對象名冊（僅限自己開的營），呼應訂閱 schema 預留的 `mentor:students` |
| **內容層** | 打卡、心得、營內互動 | 營期間可見＋期滿唯讀 90 天，之後消失 |
| **現況層** | 學員離營後的任何活動、其他 practice | **永不可見**——「追蹤」與「監控」的分界線 |

規則細節：

- **學員退出**：內容層即時消失（含歷史的個人視圖）；名冊層留最小紀錄（參加過、已退出）。聚合統計保留已計入的歷史數字。已寄出的週報不回收（退出後不再出現在任何**新產出**的視圖與報表）。此取捨寫進同意文案。
- **教練移除成員**（2026-07-21 定案）：效果＝強制版自助退出（practice 自動轉個人、可見性即斷、學員收中性通知）；enrollment 狀態記 `removed`（與自願 `exited` 區分），審計記操作者。
- **帳號刪除例外條款**（2026-07-21 定案）：名冊層的「永久」指**計數與結構，不是個資**。學員行使帳號刪除權時，名冊列保留但個資匿名化（顯示「已刪除的使用者・第 N 期・完成狀態」），快照統計不變——教練的營運紀錄不破洞，當事人的個資確實消失。
- **一期結束**：教練端轉唯讀（可看歷史、可期末匯出），學員後續打卡不再可見。唯讀期 90 天，期滿內容層消失、只留聚合報表與名冊層——同時是續開下一期的商業鉤子。
- **平台中介的再連繫**（Phase 2）：教練對名冊上的舊學員按「邀請參加新一期」→ 由平台發站內通知/信（學員可退訂）。教練**始終拿不到 email**，但一鍵觸及所有舊學員、不用自己維護名單也沒有個資法風險；學員收到的是有脈絡的通知（「你上過的教練開新營了」）而非陌生騷擾。
- **期末成果匯出**：匯出物離開系統（招生素材），標準高於站內可見——**逐學員 opt-in**，只匯出同意者。
- **共同帶領/助教**：可見性跟著 `cohort_members` 的 owner/assistant 角色走，人離開角色即失去存取。MVP 助教與 owner 同權，後續再分層。

## 身分模型：關係制，不加全域角色

不在 RBAC 加 teacher/coach 角色（全域角色無法表達「只能看自己的營」的範圍限制，且會產生「有角色沒營」的殭屍狀態）。改用：

- `cohort_members.role = owner / assistant / member`，權限判斷全部基於 cohort 關係（與既有 `requireOwnership` 模式一致）。
- 開營資格：**MVP＝人工開通**（依 FRD 7.1.1，organizers 由內部審核建立）；**配額制是 P3 自助化後的接替機制**（免費開 1 個 15 人內小期，付費解鎖，以 `cohorts.capacity` 承載）——兩者是階段接替，不是矛盾（2026-07-21 註記）。人工審核長期只留給「認證教練」標章等信任訊號。

## 功能規劃

### Phase 0 — Concierge 驗證（幾乎零程式碼）

找 3–5 位設計夥伴講師：講師用一般帳號建模板、課程學員用已上線的複製實踐打卡；「儀表板」由團隊每週人工/script 產出摘要寄講師；期末人工匯出一次。

**招募方式：掛在 GTM w1-w2 執行包的 B 軌 outreach 上跑，不另起爐灶**——「我們人工幫你做全班節律報告一期」本身就是 outreach 信的開信鉤子，比推銷尚不存在的產品誠實，也比訪談更能篩出真的會用的人。

**設計夥伴篩選（GTM 名單標準之上加三條）**：
1. 課程作業天然像 practice（語言每日練習、寫作週更、健身打卡等重複性練習型；一次性大作業型先不要）
2. 一個月內有新班開課（Phase 0 要觀察完整週期）
3. 熱介紹優先（體驗粗糙、人工週報可能遲到，只有關係戶撐得住）

**Go / Kill 訊號（開跑前定案，避免八週後只得到「反應好像不錯」）**：
- **Go**：≥3 位講師走完整期，且 ≥2 位在收到週報後有實際回應行為（去學員打卡下留言/鼓勵）≥2 次——驗證「看見→回應」循環真的發生，而非禮貌性看報告
- **Go（付費意願）**：期末問「下一期 NT$4,900 續嗎」，≥1 位口頭承諾或願意預付折扣價
- **Kill**：學員打卡率與講師上一期（無島島）相比無可感知差異，或多數學員打卡第三週後歸零——陪跑空間本身不成立

基準線的誠實限制：無對照組，「打卡率提升」嚴格說測不出來；n=3–5 階段**質性訪談權重高於數字**——講師願不願意把下一期也搬上來，比任何比率誠實。

### Phase 1 — MVP：教練痛點的完整解方（2026-07-16 討論改版）

**MVP 的範圍定義不是「上線日功能集」，是「第一期營從開營到期末的完整價值閉環」。** 理由：教練的三大痛（課後全盲、回應費力、招生累）必須在他付費的第一期內全部被回答，否則「不給監控」的產品立場沒有對價，教練第一期就流失。期末功能（成果匯出）在第一期第 8 週就會用到，放後面的 phase 等於讓第一批教練拿不到最強賣點。

分兩批交付（依一期營的時間軸）：

**開營就緒包**（上線日就緒——沒有這些，營開不起來）：

| 功能 | 解的痛 | 複用積木 |
|---|---|---|
| 開營、期間設定、加入方式（邀請連結/QR ＋ email 名單上傳並發邀請信，2026-07-20 定案兩者皆做） | 基礎 | 新資料模型；名單來源合法性責任明文歸夥伴 |
| 課程作業模板 → 學員一鍵複製並綁定營 | 基礎 | 複製實踐已上線，只差綁定 |
| 節律儀表板（熱度圖、標籤分佈、**共同卡點聚合**） | 課後全盲；卡點洞察回饋教學設計 | user-stats 統計模式 |
| 需要關心名單（流動中斷訊號＋最後一則打卡） | 「今天該找誰花五分鐘」 | 儀表板附帶查詢 |
| 營內動態牆＋**AI 鼓勵草稿一鍵回應** | 回應費力——把回應一位學員的成本從三分鐘降到十秒 | 靈感牆/快速回應/留言已上線；server 已有 checkin-encouragements |
| 模板頁教練簡介＋課程連結 | 招生累——反向漏斗：島島用戶瀏覽/複製模板 → 認識教練 → 報名課程 | 模板頁加欄位 |
| 自助退出／轉為個人實踐 | 學員自主權——加入靠同意、退出也自助（見可見性模型） | 解除綁定即斷可見性 |

**期末就緒包**（第一期第 8 週前必須就緒——前 7 週用不到，但期末是續費決策時刻）：

## 與既有規劃的關係

- **共同挑戰（Challenge）**：結構與陪跑營高度相似（一群人、一段期間、共同主題、群體牆、權限分層）。底層做通用群組容器（cohort），`type` 區分「共同挑戰」（官方發起）/「陪跑營」（教練發起），權限與計費在上層分岔——避免兩套資料模型。島島官方自辦的衝刺團也用同一容器。
- **Buddy**：獨立（一對一媒合），但同營學員是天然配對池，Phase 2/3 加分項。
- **「期」的語意**：營有明確起迄，**續開下一期是新 cohort**（複製設定），不是延長——這決定計費週期與匯出邊界。
- **期間與人數由教練自訂**：4–12 週、1 人到 40 人皆為合法形態——一對一短期諮詢（市場稱「陪跑計畫」）到團體營跑在同一容器上，n=1 時儀表板退化為單人節律視圖、動態牆退化為一對一往來，機制不變。產品功能、文案與定價**以團體營為設計中心**（獲客飛輪：一位教練帶 20–40 位學員進來；一對一諮詢收入好但不帶新用戶），不為一對一做專屬功能。
