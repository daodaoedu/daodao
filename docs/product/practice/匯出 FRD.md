## **Purpose**

透過將平台上的數位足跡轉化為學習者可完全掌控、永久留存的個人知識資產。

本版本將「資料攜帶權（Data Portability）」與「外置 AI 互動權（External AI Orchestration）」完全解耦（Decouple），並在 UI 介面上提供**雙重觸發入口**（頁面底部直觀按鈕、以及卡片/詳情頁的「三點選單」），以提升多場景下的操作便利性，提供純粹、結構化的 Markdown 相容 .txt 歷程檔案下載，確保用戶本地知識庫的純淨與主權。

## **Scope**

* **納入範圍**：  
  * 僅限「總結頁擁有者視角（Owner View）」之操作  
  * **入口一：總結頁底部直觀按鈕**（雙按鈕配置）  
  * **入口二：實踐卡片與詳情頁的「三點資訊選單（Kebab Menu）」**  
  * **功能一：下載歷程**：一鍵下載純文字 .txt（Markdown 相容語法，內含結構化 資訊，不含 AI 指令）  
  * **功能二：複製 AI 提示詞**：一鍵將動態編譯的 AI 提示詞複製  
* **排除範圍**：  
  * 公開/訪客視角（Public View）的下載與複製功能

## **Functional Requirements**

### **UI 入口設計與版面配置（擁有者視角）**

本功能必須提供兩種 UI 入口，兩者皆僅在「擁有者身分」驗證通過時顯示：

#### **入口一：總結頁底部歸檔區（直觀按鈕）**

* 在總結頁「擁有者視角」下方的「歷程歸檔管理」區塊，並列排版兩個主要操作按鈕：  
  * **【匯出文字檔(.txt)】**  
  * **【複製 AI Mentor 提示詞】**  
  * 按鈕上方附帶微文案：「複製 Prommpt 讓 AI 分析」

#### **入口二：三點選單隱藏入口（Kebab Menu）**

* **展示位置**：  
  1. **實踐卡片**右上角的 ... 更多操作按鈕。  
  2. **實踐詳情頁**頂部標題列右上角的 ... 更多操作按鈕。  
* **下拉選單項目**：當擁有者點擊 ... 開啟下拉選單時，選單清單中須動態插入以下兩個選項（排在「編輯實踐」、「刪除實踐」等常規操作之上）：  
  1. **【匯出文字檔(.txt)】**：點擊直接觸發 .txt 檔案下載。  
  2. **【**複製 Prommpt 讓 AI 分析**】**：點擊直接執行複製，不跳轉頁面，並觸發 Toast 提示。  
* **權限防禦**：若為遊客或非擁有者查看該卡片/頁面，下拉選單中**絕對不可**出現此兩項「匯出與複製」選項。

## **Test Points**

### **1\. UI 顯示與按鈕獨立性測試**

* **測試點 1.1（直觀按鈕）**：登入擁有者帳號進入實踐總結頁，驗證下載按鈕與複製 Prompt 按鈕是否各自獨立，微文案與圖標是否正確渲染。  
* **測試點 1.2（三點選單顯示）**：登入擁有者帳號，在 Dashboard 的實踐卡片上點擊 ...，以及進入詳情頁點擊右上角 ...，驗證下拉選單中是否正確出現兩個選項。  
* **測試點 1.3（非擁有者防禦）**：以遊客或其他使用者帳號存取同一個實踐頁面/卡片，展開其三點選單，驗證選單中**絕對沒有**出現任何與「匯出」或「複製 Prompt」相關的選項。

### **2\. 下載檔案純淨度測試**

* **測試點 2.1**：點擊三點選單中的【📥 匯出歷程文字檔 (.txt)】，下載產出的檔案，打開並驗證內文，確認完全沒有包含任何 AI 指令或 Prompt 喚醒字眼

### **3\. 剪貼簿複製與動態編譯測試**

* **測試點 3.1**：不論點擊「頁面底部複製按鈕」或「三點選單中的複製選項」，驗證剪貼簿內容中的 \[主題實踐名稱\]、\[practice\_id\] 是否皆被正確替換為該頁面的真實數據。  
* **測試點 3.2（跨平台相容性）**：在行動裝置（iOS/Android）上點擊三點選單中的複製選項，驗證是否能正常寫入剪貼簿
