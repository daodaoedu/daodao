## **Purpose**

本功能主要在解決學習社群中常見的「社交冷場」問題。透過將傳統的「按讚」升級為具備對話感的「快速回應 (Quick Reactions)」，並輔以「引導文字 (Prompts)」，達到以下目標：

1. **降低參與門檻：** 透過「鷹架作用 (Scaffolding)」，幫助不擅長表達的用戶克服「空白留言框焦慮」。  
2. **深化學習互動：** 引導用戶從單純的情緒表達（如：好厲害）轉向具備知識價值的回饋（如：我也是、學到了）。  
3. **強化同儕連結：** 利用口語化的反應標籤（如：一起加油），營造心理安全感與陪伴感。

## **Scope**

* **前端介面：** 包含主題實踐卡片和回應。  
* **互動邏輯：** 定義點擊反應後，如何連動留言框的狀態（Focus, Placeholder）。  
* **數據關聯：** 反應圖示與用戶留言的資料庫關聯與前端呈現。

## **Functional Requirements**

### **回應清單**

系統應提供以下4種反應：

| 口語標籤 (Label) | 英文 |
| ----- | ----- |
| **加油** | Support |
| **啟發** | Insightful |
| **共嗚** | Relate |
| **好奇** | Curious |

### **核心互動流程**

1. **觸發行為：** 用戶點擊反應按鈕（或於行動端長按展開選單）  
2. **狀態同步：** 系統將該反應標記為「選取」，計數器即時更新。  
3. **單獨留言：**使用者可僅留言，不使用快速回應鍵

### **顯示規格**

* **按鍵顯示：**快速反應及留言按鈕同時顯示在最外層卡片，並可直接點選作用  
* **計數：**反應與留言數量需分別顯示  
* **聚合顯示：** 若同一則內容有多人按同類反應，應顯示「💡 \[User A\] 與其他 5 人學到了」。  
* **個人化反饋：** 用戶在留言列表中，自己的留言旁應清晰標註所選的反應圖示。

###

### **留言與對話系統 (Comments)**

* **層級限制**：留言限二層回應，以保持討論的扁平化與易讀性。  
* **標記功能 (@mention)**：支持在留言中透過 `@` 喚起用戶清單並進行標記，當用戶打`@ 時可自動帶出留言區用戶選單`

## **Test Points**

### **互動準確性測試**

* 驗證點擊 4 種不同反應時，留言框是否能正確顯示對應的「引導文字」。  
* 驗證「自動聚焦 (Auto-focus)」功能在 Web 與 Mobile 端是否皆能順利執行且無破版。  
* 確認連續切換不同反應時，留言框的 Placeholder 是否能即時更新而不產生舊文字殘留。

### **資料持久性測試**

* 測試在「未輸入留言」的情況下送出反應，重整頁面後反應計數是否正確。  
* 測試在「有輸入留言」的情況下送出，該留言是否能與正確的反應圖示（如 🙌）正確綁定。

### **壓力與異常測試**

* 當用戶快速連續點擊多個反應時，系統是否能正確處理最後一次點擊的狀態。  
* 測試在網路延遲 (Poor Connection) 情況下，反應計數與留言框聚焦的先後順序是否會造成使用者困惑。  
* 驗證長按選單（行動端）在不同瀏覽器（Safari, Chrome）上的觸發靈敏度
