# A4 Build-in-public Builder

公開記錄自己進度的創作者／side project builder／電子報作者。學習與建造本身已是習慣，不需要平台提供動機。

動機結構：需要的是**輕量的公開記錄容器與同類的可見性**——build in public 的核心報酬是被同路人看見與回應。對工具遷移成本敏感（已有 Threads/電子報等既有記錄管道）。

GTM 角色：創世聚落原型用戶（如 [[org-pro-ai-news]]）——不是被獲取的 TA，而是最早願意公開使用並替島島說話的種子住民。不適用標準 B 軌 partner 漏斗。

痛點驗證（P2 完成，關鍵題成立）：「既有管道（Threads）哪裡不夠用」已由兩筆獨立第一手證據回答＝洗版、找不回、無法回顧，新建 [[pain-progress-buried-in-feed]]。證據：破萬追蹤的公開日更寫作者自述「一天發 15–20 篇根本就在大洗版」、收藏「沒有太好的做法只能一則一則翻找」（[[signal-jemmy-threads-wash-and-lost]]）；Threads 經營講師教學文「之前寫過的文就是找不到、發完就石沉大海、沒有分類功能」（[[signal-richsong-cant-find-old-threads]]）。故 confidence 升為 evidenced——此 persona 對島島有真需求。

研究欠債（剩餘＋風險）：仍只有 1 個 evidenced PainPoint。更關鍵的是採用意願未驗證：痛點成立 ≠ 會遷移——A4 對工具遷移成本敏感（已有 Threads／電子報既有管道），島島能否成為其記錄容器，需創世聚落原型（如 [[org-pro-ai-news]]）實測。
