# 公開記錄的進度在動態流裡被洗掉、之後找不回、無法回顧

Build-in-public builder／公開記錄型創作者（[[persona-a4]]）用 Threads 等社群記錄進度時：高頻發文自成「大洗版」，累積的內容缺乏分類與可回溯結構，之後既找不回、也無法回顧成一條可見的成長軌跡（「發完就石沉大海」）。

TA 原話語言：
- 「我常在一天內發 15–20 篇……根本就在『大洗版』。」；收藏「沒有太好的做法……只能一則一則翻找」（[[signal-jemmy-threads-wash-and-lost]]）
- 「之前寫過一篇很詳細的貼文，但就是找不到」「Threads 沒有像部落格那樣的分類功能」「每天發文要往回找兩個月前的貼文，會滑到懷疑人生」「發完就石沉大海」（[[signal-richsong-cant-find-old-threads]]）

兩筆獨立來源成立此痛，回答 persona-a4 的關鍵待驗證問題「既有管道（Threads）哪裡不夠用」＝洗版、找不回、無法回顧。島島的答案是把 ephemeral 的貼文換成結構化打卡紀錄（[[feature-checkin]]）、可回顧與匯出的學習歷程（[[feature-export]]），並在靈感牆被同路人看見（[[feature-inspire-feed]]）。注意：A4 對工具遷移成本敏感（已有 Threads／電子報），此痛成立不代表會遷移——採用意願仍待原型驗證。solved_today_by 暫留空：Threads-as-記錄容器尚無對應 Competitor（kind: diy）節點。
