## **Purpose**

本文件旨在定義儀表板「探索相關主題」區塊的功能規格。透過移除原有的「已完成」靜態區域，轉而導入動態推薦系統，提升使用者與產品內容的互動率，並建立正向與負向的回饋機制以優化推薦品質。

## 

## **Scope**

* 儀表板首頁 UI 佈局調整（移除舊區域、新增新區域）  
* 推薦卡片元件的開發  
* 推薦理由的顯示邏輯

## 

## **Functional Requirements**

### **頁面佈局調整**

* **移除舊區塊**：系統需在儀表板首頁中移除原有**固定的**「已完成 」區塊。  
* **「全部」區域顯示**：避免列出所有主題實踐擠壓到「探索相關主題」，可預設顯示進行中及未開始，並增加「顯示更多」點選後看到全部內容  
* **新增探索區塊**：在主題實踐區塊下方新增「探索相關主題」區塊。預設顯示最多3個推薦，若有更多則顯示「查看更多推薦內容 」讓使用者點選  
* **AI 使用聲明**：說明此「此區域 AI 生成推薦」

### **推薦卡片元件規格**

每張推薦卡片必須包含以下資料欄位：

* **推薦理由 (Match Reason)**：顯示一段文案，說明推薦此主題的原因  
  * 基於你的「專業領域」  
  * 基於你「想探索的領域」  
  * 基於你目前進行的主題實踐  
  * 你跟這位夥伴興趣相近

* **主題資訊**：可與目前「靈感」顥示相同  
* **互動元件**：包含「👍 」與「👎 」按鈕

### **互動與狀態管理**

* 👍**互動**：  
  * 使用者點擊👍後，按鈕狀態變更為 Active（填色），系統需記錄使用者喜歡  
  * 再次點擊可取消喜歡，狀態恢復原樣  
* 👎**互動**：  
  * 點擊👎按鈕，前端應立即顯示「確認隱藏？」對話框。  
  * 使用者確認後，系統需記錄使用者不喜歡，並執行卡片隱藏動畫

### **推薦根據**

新註冊用戶可以1,2,3為主，隨著使用者在平台上活動提升，增加4及5，並參考6，調整

1. 專業領域  
2. 想探索的領域  
3. 有相似「專業領域」或「想探索的領域」的用戶  
4. 自己主題實踐的內容  
5. 瀏覽的內容  
6. 使用者對於推薦內容的互動👍/👎

### **資料對接要求**

* **持久化記錄**：系統需確保隱藏記錄與使用者帳號綁定，使用者於不同裝置登入時，不應再看到已隱藏的推薦

###  **空狀態邏輯** 

* **觸發條件**：當無符合推薦內容或前端經由使用者「不喜歡」操作導致當前顯示列表長度為 0 時  
* **呈現**：  
  * 隱藏原本的卡片佈局  
  * 包含一組指定的圖示、標題、描述文字與 CTA 按鈕  
  * 描述文字: 你可以先專注於當前的計畫，或到「靈感」分頁看看其他主題實踐  
* **CTA 點擊行為**：系統需執行 Tab 切換至「靈感」分頁。

## 

## **Test Points**

### **正向測試** 

* **佈局確認**：進入儀表板，確認「已完成」區塊已消失，且「探索相關主題」區塊正確顯示在指定位置。  
* **推薦資料呈現**：確認推薦卡片能正確顯示標題、作者頭像、推薦理由等欄位，文字無截斷問題（除描述外）。  
* **喜歡功能測試**：點擊喜歡按鈕，確認 UI 狀態改變，且後端資料庫確實收到該使用者對該專案的 Positive 標記。

### **負向與異常測試** 

* **隱藏機制測試**：點擊不喜歡並確認隱藏後，重新整理頁面，該張卡片必須不再出現。  
* **資料耗盡測試**：當所有推薦內容都被點擊「不喜歡」後，測試 UI 是否能優雅地顯示提示
