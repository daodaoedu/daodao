# 註冊, 驗證及歡迎信

## **Purpose**

核心策略為將「行政填表」轉化為「解鎖儀式」，協助用戶在 5 分鐘內建立帳號並獲得系統推薦的首個「主題實踐 (Practice)」。

## **Scope**

新用戶首次登入後，需完成 Onboarding 流程才能使用完整功能。

## **Functional Requirements**

### Step 1：個人資訊

**目的**：建立基本個人檔案並確認使用者年齡

| 欄位 | 必填 | 說明 | 驗證規則 |
| :---- | :---- | :---- | :---- |
| Email | ✅ | 從 OAuth 提供者帶入，不可修改 | \- |
| 名字 | ✅ | 顯示名稱 | 最多 50 字 |
| 生日 | ✅ | 用於年齡驗證 | 需滿 16 歲 |
| 帳號 ID | ✅ | 個人專屬網址 | 3-15 字，僅英數字 |

### Step 2：興趣領域

**目的**：了解用戶興趣，提供個人化推薦

| 欄位 | 必填 | 說明 |
| :---- | :---- | :---- |
| 專業領域 | ✅ | 至少選 1 個，最多 5 個 |
| 興趣領域 | ✅ | 至少選 1 個，最多 5 個 |

### Step 3：來源調查

**目的**：了解用戶如何得知平台，用於行銷分析

| 欄位 | 必填 | 說明 |
| :---- | :---- | :---- |
| 來源管道 | ✅ | 單選 |
| 其他說明 | ✅ | 選擇「其他」時必填 |

### Step 4：完成註冊頁面

**目的**：確認註冊成功，引導至 email 驗證

### Step 5：完成驗證及歡迎信

**目的**：確認驗證成功

* 完成驗證頁面引導至設定頁面([https://app.daodao.so/en/settings），完成偏好設定及個人資訊](https://app.daodao.so/en/settings），完成偏好設定及個人資訊)  
* 引導文案「太棒了！完成個人設定，獲得更精準的學習推薦。」  
* 依照不同來源寄出歡迎信 [Welcome Letter \- 202601 \- Google 文件](https://docs.google.com/document/d/11raDGrgIu5pdjcJzqwhsmWi10omLCsj1nmWbkoAcmhE/edit?tab=t.9otkgldskpce)

# 設定頁面完整度引導

## **Purpose**

本功能旨在降低使用者在「設定」頁面中的行政負擔感。透過即時的進度顯示與動態提示（微文案），引導使用者完成「帳號設定」、「偏好設定」與「公開資訊」的補完，並強調補完資料的實質好處。

## **Scope**

涵蓋設定頁面內的三個資料區塊：

* **偏好設定** (如：時數投入、社交能量)  
* **帳號設定** (如：頭像、生日)  
* **公開資訊** (如：Tagline、About Me)

## **Functional Requirements**

### **視覺化進度 (Completion Progress)**

* **顯示邏輯：**  
  * 「設定」側邊欄：顯眼處採用「完成數 / 總項目數」（例如：`2/3`）。  
* **計算權重：**  
  * 用戶完成註冊，預設顯示1/4  
  * 當上述任一資料區塊從變更為「已填寫」狀態時，進度分子應即時增加。  
  * 每完成一個資料區塊儲存，如**偏好設定，**進度分子增加至2/4，以此類推。

### **動態引導標籤與文案 (Dynamic Labels & Micro-copy)**

* **觸發條件：**  
  * 在點選進入設定頁面時，上方需顯示動態標籤。  
* **微文案內容：**  
  * 統一使用價值導向文案：**「完成個人資訊設定，獲得更精準的學習推薦」**。  
* **隱藏邏輯：**  
  * 一旦使用者完成輸入有效內容並儲存，該標籤應立即消失。

### **資料區塊頁**

* 未完成區塊需有視覺化提示  
* 資料區塊內必填資料需都有資料才可儲存  
* 無資料欄位於點選儲存時需提示填寫

## **4\. 測試要點 (Test Points)**

### **功能邏輯測試 (Logic Testing)**

* 驗證進度條的分子是否與「已填寫之項目」數量完全一致（例如：填寫 1 個變 1/4，再填一個變 2/4）。  
* 每一個資料區塊內資料是否在儲存後確實存入

### **UI/UX 表現測試 (UX Testing)**

* 驗證「動態標籤」在手機版螢幕上是否會導致版面錯位（Responsive Check）。
