# 打卡 Agent 輔助（引導 + 週期小結 + 斷鏈接回）

> 產出自 `agent-ux-design` skill 五步驟引導（2026-07-11）。介入模式：**C + A 組合**。

## 1. 使用者意圖

讓使用者**持續執行、感覺到累積**——不斷鏈，而且回頭看得到自己走了多遠。打卡不是「填完表單」，是每天留下一塊之後拼得起來的碎片。

## 2. 現況苦工

| 苦工點 | 類型 | 為什麼累 |
|---|---|---|
| 心得不知寫什麼 | 空白恐懼 | 描述欄常留空或流水帳，累積下來的痕跡沒有料、回顧時無感 |
| 斷鏈後就放棄 | 動能斷裂 | 漏打一兩天後罪惡感 → 乾脆不回來；沒有「接回來」的機制 |
| 打很多天看不到累積 | 累積無感 | 每天打卡都是單點，沒有東西把碎片串成「你已經走了多遠」 |

現況參考：打卡 Sheet 為單頁表單（心情選填 → 標籤必填 → 描述選填 600 字 → 照片選填 3 張）。既有輕量輔助：靜態 tag prompt（`use-tag-prompt.ts`）、打卡成功鼓勵語（`encouragement`）、靜態反思問題 25 題（`use-reflection-question.ts`，預設關閉）。

## 3. Agent 介入設計（模式 C + A）

### 3a. 打卡當下：情境化引導問句（模式 C）

**UI 流程**
1. 使用者照舊選心情、選標籤
2. 描述欄上方出現 **1 題貼身引導問句或起頭句**（取代／升級現有靜態 tag prompt 與 25 題反思問題）
3. 「換一題」按鈕可重生；也可完全無視直接自己寫
4. 使用者接著引導句往下寫 → 完成打卡

**後端包裝**
- 大 prompt 模板：`你是實踐陪伴引導者…實踐名稱: {practiceName}／行動描述: {actionDescription}／今天第 {dayN} 天（共 {totalDays} 天）／最近 2-3 次打卡摘要: {recentCheckIns}／今日標籤: {tags}／心情: {mood} → 產出 1 題不超過 30 字的引導問句或起頭句`
- 控制邊界：只生成問句／起頭句，不代寫心得本文；長度上限；語氣規範由 skill 寫死
- 可沿用 `daodao-ai-backend`，前端掛在 `check-in-sheet.tsx` 的 description 區塊（取代 `use-tag-prompt` 的靜態注入點）

### 3b. 每 7 天／檢核點：週小結草稿（模式 A）

**UI 流程**
1. 每滿 7 天（或實踐檢核點），系統產生小結草稿並**推播提醒**「本週小結草稿好了」
2. 使用者進來看到草稿：本週打卡數、標籤／心情分布、從 check-ins 心得萃取的一段敘事小結
3. 可整段重生、可手動編輯、可放棄不發布——**不確認就不發布**
4. 確認後發布 → 成為歷程時間軸上的**里程碑卡片**（特殊樣式，插在 check-in 流之間），與覆盤的「置頂總結」定位區隔，並成為未來匯出／實踐總結頁的素材

**後端包裝**
- 大 prompt 模板：`彙整以下 {days} 天打卡紀錄…{checkInList}（每筆含日期/標籤/心情/心得）→ 產出 150 字內第二人稱小結，聚焦變化與累積，不評價、不說教`
- 控制邊界：只用該使用者自己的 check-in 內容為素材，不引入外部資訊；字數上限；未確認 = 草稿狀態
- 排程：server 端 cron 依實踐 startDate 計算週界；推播沿用既有通知機制

### 3c. 斷鏈回歸：接回訊息（模式 C）

**UI 流程**
1. 使用者漏打 ≥2 天後回來打卡，Sheet 頂部顯示一句**非懲罰式接回訊息**（reframe 中斷：「休息 3 天回來，比連續 3 天更難得」）
2. 純顯示、不需操作、不擋流程；打卡成功後的鼓勵語同步強化「回來了」

**後端包裝**
- 可併入 3a 的引導 prompt（多帶 `gapDays` 參數），或擴充既有 `encouragement` 機制的情境分支
- 控制邊界：只在 gapDays ≥ 2 時觸發；語氣白名單（不出現「可惜」「中斷」等負面詞）

## 4. 可控性檢查

- [x] 每個 agent 動作前後都有使用者可介入點：引導句可換可無視；小結是草稿、確認才發布；接回訊息純顯示
- [x] 結果落在使用者預期內：小結素材 100% 來自自己的 check-ins；引導句範圍被實踐內容框住
- [x] 可「重生／微調單格／退回上一步」：換一題、小結整段重生或手動改、不發布
- [x] 不是右邊一個 chat 視窗，是融進既有打卡 Sheet 與歷程時間軸
