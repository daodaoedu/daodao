# 建立主題實踐：逐欄小助手

> 產出自 `agent-ux-design` skill 五步驟引導（2026-07-11）。介入模式：**C（流程中小提示／自動填寫）**。

## 1. 使用者意圖

**順利「開始」一個做得下去的挑戰**。建立主題實踐是整個實踐旅程的起點，最大阻力是不知道內容要定什麼——不是不想做，是把「我想學日文」翻譯成「每天做得到的具體行動」這步卡住了。

## 2. 現況苦工

| 苦工點 | 類型 | 為什麼累 |
|---|---|---|
| 行動描述定不出來 | 空白恐懼 | 模糊念頭要濃縮成 50 字內每天做得到的具體行動，最多人卡在 Step 1 |
| 模板通用、跟自己情況有落差 | 領域門檻 | 模板預填後還是要大改，改的過程等於重新面對空白恐懼 |
| 標籤／資源不知放什麼 | 選擇焦慮 | Step 4 對新手來說不知道有什麼用、該放什麼 |

現況參考：手動建立為 5 步精靈表單（`practice/create/manual/`），欄位含名稱、行動描述（≤50 字）、日期／天數／頻率、時長／時段、標籤／資源、預覽。已有 `suggestTags` API（`tag-edit-content.tsx`）。

## 3. Agent 介入設計（模式 C：表單結構不變，在卡點塞小助手）

### UI 流程

1. **Step 1 名稱 + 新增選填小格「你為什麼想做這件事？」**
   - 提示文字：「一句話就好，例如：想看懂日文動畫、想在年底前跑完半馬」
   - 這格是 sub-prompt slot，讓後續所有提案更貼身；留空也能運作（退回只用名稱）
2. **Step 1 行動描述欄旁「幫我想」按鈕**
   - 依「名稱＋為什麼想做」提案 **3 個具體行動描述**（各 ≤50 字）供點選帶入
   - 可「再想一次」重生 3 個；點選後仍可手動編輯
3. **Step 2–3 頻率／天數／時長：預選建議值＋一句理由**
   - 依已選的行動描述，agent 預選建議選項並附一句理由（例：「背單字建議每週 4–7 天，短高頻比長低頻容易養成」）
   - 使用者可直接改選其他值，建議只是預設焦點
4. **Step 4 標籤：自動帶入建議標籤**
   - 依行動描述自動呼叫既有 `suggestTags` 帶入 2–3 個，可增刪
5. **Step 5 預覽確認**（不變）：所有 agent 帶入的值在此一覽，逐欄可退回修改

### 後端包裝

- 大 prompt 模板（行動描述提案）：`你是習慣養成教練…實踐名稱: {name}／動機: {motivation}／限制: 行動描述須 ≤50 字、每天可獨立完成、動詞開頭、可量化 → 產出 3 個由易到難的行動描述`
- 大 prompt 模板（頻率建議）：`行動描述: {actionDescription} → 從固定選項（頻率 2-4/3-5/4-7 天、時長 15/30/45/60 分）中挑建議值，附 ≤25 字理由`
- 由哪個 Skill／Agent 包裝：`daodao-ai-backend` 新增 practice-create-helper 端點；標籤沿用既有 `suggestTags`
- 控制邊界：行動描述受 schema 50 字限制；頻率／時長**只能從既有表單選項中挑**（不生成表單外的值）；提案數固定 3 個避免選擇焦慮

### 與模板路線的關係

模板建立路線保持不變；逐欄小助手掛在共用的 5 步表單上，因此**從模板進來的使用者改欄位時同樣享有「幫我想」**，間接緩解「模板通用、有落差」的苦工（改模板不再面對空白）。

## 4. 可控性檢查

- [x] 每個 agent 動作前後都有使用者可介入點：提案是「供選」不是「代填」；預選值可直接改；標籤可增刪
- [x] 結果落在使用者預期內：提案基於他自己填的名稱＋動機；頻率／時長只從既有選項挑
- [x] 可「重生／微調單格／退回上一步」：「再想一次」逐欄重生；5 步精靈本身支援回上一步；Step 5 總覽可回改
- [x] 不是右邊一個 chat 視窗，是融進既有 5 步表單
